
### Конспект лекции: Композиционные (комитетные) алгоритмы

**Главная идея:** Один классификатор может ошибаться, но если объединить несколько классификаторов ("экспертов") и согласовать их мнения (например, голосованием), итоговое решение будет **более точным и устойчивым**, чем решение каждого по отдельности.

**Ключевая метафора:** "Одна голова хорошо, а две — лучше". Комитет экспертов принимает более взвешенное решение, чем один эксперт.

---

### Введение: Зачем нужны композиции?

| Один классификатор | Комитет классификаторов |
|--------------------|--------------------------|
| Может переобучиться под конкретную выборку | Ансамбль сглаживает переобучение |
| Чувствителен к шуму и выбросам | Устойчивее к шуму |
| Может иметь систематическую ошибку | Разные алгоритмы компенсируют ошибки друг друга |
| Ограниченная обобщающая способность | Часто даёт лучшее обобщение |

**Классификация классификаторов по качеству:**
- **Плохой:** вероятность ошибки $p_g \le 1/2$ (хуже монетки)
- **Слабый:** $p_g = 1/2 - \varepsilon$ (немного лучше монетки)
- **Сильный:** $p_g = \varepsilon$ (очень маленькая ошибка)

**Цель бустинга:** собрать из **слабых** классификаторов один **сильный**.

### Глава 1. Общая структура композиционных алгоритмов

**Формальное описание:**

1. **Базовые алгоритмы (БА):** $g_k(x)$, $k = 1..L$ — каждый даёт свою оценку.
2. **Корректирующая операция:** $G: \mathbb{R}^L \to \mathbb{R}$ — объединяет оценки БА.
3. **Решающее правило:** $G': \mathbb{R} \to D$ — принимает итоговое решение.

**Итоговая композиция:**
$$G'(x) = G'[G(g_1(x), ..., g_L(x))]$$

#### 1.1. Линейные композиции (самые распространённые)

$$G(x) = \sum_{k=1}^{L} \alpha_k g_k(x), \quad \sum_{k=1}^{L} \alpha_k = 1, \quad \alpha_k \ge 0$$

- $\alpha_k$ — веса, учитывающие вклад каждого БА.
- **Простое голосование:** $\alpha_k = 1/L$ — все эксперты равноправны.

**Для двух классов ($\omega_1$ и $\omega_2$, метки $+1$ и $-1$):**
$$G'(x) = \text{sign}\left(\sum_{k=1}^{L} \text{sign}(g_k(x))\right)$$

Это "голосование большинством": сколько БА сказали "+1", сколько "-1", побеждает тот, у кого больше голосов.

### Глава 2. Два главных подхода к построению композиций

| Характеристика | Баггинг (Bagging) | Бустинг (Boosting) |
|----------------|-------------------|---------------------|
| **Способ обучения** | Параллельный (все БА обучаются независимо) | Последовательный (каждый следующий учится на ошибках предыдущих) |
| **Зависимость БА** | Слабозависимые (разные подвыборки) | Сильнозависимые (веса объектов меняются) |
| **Цель** | Снизить дисперсию (устойчивость) | Снизить смещение (улучшить качество) |
| **Пример** | Случайный лес (Random Forest) | AdaBoost, LogitBoost, GentleBoost |
| **Типичный БА** | Деревья решений | Деревья малой глубины, пни (stumps) |

**💎 Суть отличия:**
- **Баггинг** — обучаем много независимых экспертов на разных подвыборках, потом усредняем. Эксперты "не завидуют" друг другу.
- **Бустинг** — обучаем экспертов по очереди. Следующий эксперт "учится на ошибках" предыдущего, получая больше внимания к сложным примерам.

### Глава 3. Баггинг (Bootstrap Aggregating)

**Название:** сокращение от *bootstrap aggregating*.

**Основная идея:** создать много разных обучающих выборок из исходной с помощью **бутстрепа** (случайная выборка **с возвращением**), обучить на каждой свой классификатор, а затем объединить их голосованием.

#### 3.1. Бутстреп (Bootstrap)

- Из исходной выборки $X^N$ (размер $N$) делаем случайную выборку **с возвращением** того же размера $N$.
- В среднем такая выборка содержит **≈63%** уникальных исходных объектов (остальные — повторения).
- Оставшиеся **≈37%** не попавших в выборку объектов называются **Out-Of-Bag (OOB)**.

#### 3.2. Деревья решений (Decision Trees)

**Что это:** Граф, где в каждой вершине проверяется условие по одному из признаков ($x_j < \text{порог}$), и в зависимости от результата мы идём налево или направо. Листья — решение (класс).

**Ключевые понятия:**
- **Загрязнённость вершины (impurity):** показывает, насколько "смешаны" классы в этой вершине.
- **Индекс Джини (Gini index):** $J(t) = 1 - \sum_{i=1}^{M} p_i^2$, где $p_i$ — доля класса $i$ в вершине.
  - $J = 0$ — вершина "чистая" (один класс).
  - $J$ максимален — классы перемешаны поровну.
- **Расщепление (split):** выбор признака и порога, которые максимально уменьшают загрязнённость.

**Проблемы деревьев решений:**
- **Переобучение:** полное дерево идеально классифицирует обучающую выборку, но плохо работает на новых данных.
- **Решения:** остановка расщепления (stop early) или **усечение (pruning)** — отрезание лишних ветвей.

#### 3.3. Случайный лес (Random Forest) — главный представитель баггинга

**Как строится:**
Для каждого $k = 1..L$ (каждого дерева в лесу):
1. Формируем бутстреп-выборку из исходных данных.
2. Строим дерево решений **без усечения** (полное дерево), но:
   - В каждой вершине рассматриваем не все $n$ признаков, а **случайное подмножество** из $p$ признаков (обычно $p \approx \sqrt{n}$).
   - Лучший признак выбираем только среди этих $p$.

**Итоговая классификация:** голосование по всем $L$ деревьям.

**Преимущества случайного леса:**
- **Двойная случайность** (бутстреп + случайные признаки) → деревья слабо коррелированы → ансамбль устойчив.
- **Не требует усечения** (pruning не нужен).
- **Практически не переобучается** (при большом $L$ ошибка на тесте монотонно снижается).
- **Простая настройка:** всего 2 параметра ($L$ и $p$).
- **OOB-оценка качества:** можно оценить ошибку без отдельной тестовой выборки (прогнать OOB-объекты по деревьям, которые их не видели).

**Результаты из лекции (рис. 3):** случайный лес значительно превосходит одиночное дерево решений на примере "заплетённые восьмёрки".

### Глава 4. Бустинг (Boosting)

**Название:** *boosting* — "усиление".

**Основная идея:** последовательно добавлять классификаторы, каждый из которых фокусируется на тех объектах, где предыдущие классификаторы ошиблись.

#### 4.1. Ключевой механизм — перевзвешивание объектов

- На каждой итерации у каждого объекта есть **вес** $w_i$ (важность).
- Изначально все веса одинаковые ($w_i = 1/N$).
- Классификатор обучается так, чтобы **минимизировать взвешенную ошибку**:
  $$E(g_k, w) = \sum_{i=1}^{N} w_i \cdot [g_k(x^{(i)}) \neq d^{(i)}]$$
- После обучения веса ошибочно классифицированных объектов **увеличиваются** (чтобы следующий классификатор обратил на них внимание).
- Веса правильно классифицированных объектов **уменьшаются**.
- Веса нормализуются (сумма = 1).

#### 4.2. Алгоритм AdaBoost (Adaptive Boosting)

**Дано:** обучающая выборка $XD^N = \{(x^{(i)}, d^{(i)})\}$, $d^{(i)} \in \{-1, +1\}$.

**Шаги:**
1. Инициализация: $w_i = 1/N$, $i = 1..N$.
2. Для $k = 1$ до $L$:
   - Обучить базовый классификатор $g_k(x)$, минимизирующий $E_k = \sum_{i} w_i [g_k(x^{(i)}) \neq d^{(i)}]$.
   - Если $E_k = 0$ или $E_k \ge 1/2$, остановиться.
   - Вычислить вес классификатора: $\alpha_k = \frac{1}{2} \ln\left(\frac{1 - E_k}{E_k}\right)$.
   - Обновить веса объектов: $w_i \leftarrow w_i \cdot \exp(-\alpha_k d^{(i)} g_k(x^{(i)}))$.
   - Нормализовать веса: $w_i \leftarrow w_i / \sum_{j} w_j$.
3. Итоговый классификатор: $G(x) = \text{sign}\left(\sum_{k=1}^{L} \alpha_k g_k(x)\right)$.

**Анализ:**
- $\alpha_k$ тем больше, чем меньше ошибка $E_k$ (хорошие эксперты имеют больший голос).
- Множитель $\exp(-\alpha_k d^{(i)} g_k(x^{(i)}))$:
  - Если классификатор угадал ($d^{(i)} g_k(x^{(i)}) = +1$), вес объекта умножается на $e^{-\alpha_k} < 1$ (уменьшается).
  - Если ошибся ($d^{(i)} g_k(x^{(i)}) = -1$), вес умножается на $e^{+\alpha_k} > 1$ (увеличивается).

#### 4.3. Разновидности бустинга в MATLAB

| Метод | Особенность |
|-------|--------------|
| `AdaBoostM1` | Классический AdaBoost для двух классов |
| `AdaBoostM2` | Обобщение на много классов |
| `LogitBoost` | Использует логистическую функцию потерь |
| `GentleBoost` | Квадратичная функция потерь, мягче реагирует на выбросы |
| `RobustBoost` | Устойчив к выбросам (не даёт весам отдельных объектов слишком расти) |

На примере "заплетённые восьмёрки":
- **AdaBoostM1** и **GentleBoost** оба дают хорошее качество.
- Бустинг значительно улучшает качество одного дерева (сравните с рис. 2 для одиночного дерева).

### Глава 5. Сравнение баггинга и бустинга (сводная таблица)

| Параметр | Баггинг (Random Forest) | Бустинг (AdaBoost) |
|----------|-------------------------|---------------------|
| **Параллелизм** | Высокий (легко распараллелить) | Последовательный (каждый следующий зависит от предыдущего) |
| **Переобучение** | Почти не переобучается при росте $L$ | Может переобучиться при слишком большом $L$ |
| **Чувствительность к шуму** | Относительно устойчив | Чувствителен (шумовые объекты получают большие веса) |
| **Сложность настройки** | Простая (2 параметра) | Требует выбора $L$ и типа БА |
| **Интерпретируемость** | Средняя (лес из многих деревьев) | Низкая (композиция многих классификаторов) |
| **Типичный БА** | Полные деревья решений | Пни (деревья глубины 1), слабые классификаторы |
| **Скорость обучения** | Средняя (можно распараллелить) | Средняя (последовательная) |
| **Скорость классификации** | Высокая ($L$ деревьев) | Высокая |

### Итог по композиционным алгоритмам

1. **Главная идея:** ансамбль классификаторов часто работает лучше, чем любой отдельный классификатор.

2. **Баггинг (Bagging):**
   - Обучаем много независимых классификаторов на разных бутстреп-выборках.
   - Итог — голосование или усреднение.
   - Снижает **дисперсию** (переобучение).
   - **Случайный лес** — золотой стандарт баггинга.

3. **Бустинг (Boosting):**
   - Обучаем классификаторы последовательно.
   - Каждый следующий фокусируется на ошибках предыдущих (перевзвешивание объектов).
   - Снижает **смещение** (систематическую ошибку).
   - **AdaBoost** — классический алгоритм.
   - Может переобучиться, чувствителен к шуму.

4. **Деревья решений:**
   - Интуитивно понятны, легко интерпретируемы.
   - Без усечения склонны к переобучению.
   - Служат отличными базовыми классификаторами в ансамблях.

5. **Практические рекомендации:**
   - Если нужна высокая точность и данные не слишком зашумлены — попробуйте **Random Forest** (работает хорошо "из коробки").
   - Если данных мало, а нужно максимально использовать информацию — попробуйте **бустинг** (но осторожно с переобучением).
   - Если нужно быстро и просто — **одиночное дерево решений** (CART).
   - Если данные очень зашумлены — **RobustBoost** может быть лучше AdaBoost.
