### Конспект лекции №10: Метод опорных векторов (Support Vector Machine — SVM)

**Главная идея:** SVM ищет не любую разделяющую линию, а **лучшую** — ту, которая находится на максимальном расстоянии от ближайших точек обоих классов. Это называется **максимизацией зазора (margin)**. Чем шире "полоса" между классами, тем увереннее классификация.

---

### Введение: Почему не любая разделяющая прямая хороша?

Если классы линейно разделимы, существует **много** гиперплоскостей, которые их разделяют. Какая из них лучше?

**Интуиция:** Представьте реку, разделяющую два берега. Чем шире река, тем меньше вероятность, что человек с одного берега случайно окажется на другом.

В SVM мы строим не просто границу, а **полосу** между классами, и делаем её **максимально широкой**. Граница проходит **посередине** этой полосы.

---

### Глава 1. Формализация линейно разделимого случая

Пусть у нас есть два класса. Обозначим:
- $x^{(i)} \in \mathbb{R}^n$ — вектор признаков
- $d^{(i)} \in \{+1, -1\}$ — метка класса ($+1$ для $\omega_1$, $-1$ для $\omega_2$)

**Уравнение разделяющей гиперплоскости:**
$$g(x) = w^T x - b_0 = 0$$

Здесь:
- $w$ — вектор весов (перпендикулярен гиперплоскости)
- $b_0$ — сдвиг (расстояние от начала координат до гиперплоскости, нормированное на $\|w\|$)

**Правило классификации:**
$$g'(x) = \text{sign}(g(x)) = \text{sign}(w^T x - b_0)$$

#### 1.1. Условие разделимости

Для всех точек выборки должно выполняться:
$$d^{(i)} (w^T x^{(i)} - b_0) > 0 \quad \text{(строгое разделение)}$$

Для удобства мы можем **нормировать** $w$ и $b_0$ так, чтобы для ближайших к границе точек выполнялось:
$$d^{(i)} (w^T x^{(i)} - b_0) \ge 1$$

#### 1.2. Ширина разделяющей полосы (margin)

Из рисунка в лекции: расстояние между двумя граничными гиперплоскостями ($d^{(i)} g(x) = 1$ и $d^{(i)} g(x) = -1$) равно:
$$\Delta = \frac{2}{\|w\|}$$

**Вывод:** Чтобы максимизировать ширину полосы, нужно **минимизировать** $\|w\|^2$.

#### 1.3. Задача оптимизации

Мы приходим к задаче квадратичного программирования:
$$
\begin{cases}
\frac{1}{2} w^T w \to \min\\
d^{(i)}(w^T x^{(i)} - b_0) \ge 1,\quad i = 1, N
\end{cases}
$$

#### 1.4. Решение через метод множителей Лагранжа

Строим функцию Лагранжа:
$$L(w, b_0, \lambda) = \frac{1}{2} w^T w - \sum_{i=1}^{N} \lambda_i [d^{(i)}(w^T x^{(i)} - b_0) - 1]$$

Условия оптимальности (производные равны нулю):
$$\frac{\partial L}{\partial w} = w - \sum_{i=1}^{N} \lambda_i d^{(i)} x^{(i)} = 0 \quad \Rightarrow \quad w = \sum_{i=1}^{N} \lambda_i d^{(i)} x^{(i)}$$
$$\frac{\partial L}{\partial b_0} = \sum_{i=1}^{N} \lambda_i d^{(i)} = 0$$

#### 1.5. Опорные векторы

Векторы $x^{(i)}$, для которых $\lambda_i > 0$, называются **опорными**. Именно они определяют положение разделяющей гиперплоскости. Остальные точки можно удалить — решение не изменится!

**Важный факт:** $w$ — это линейная комбинация **только опорных векторов**.

#### 1.6. Окончательное решающее правило

$$g'(x) = \text{sign}\left( \sum_{i=1}^{N} \lambda_i d^{(i)} (x^T x^{(i)}) - b_0 \right)$$

### Глава 2. Случай линейной разделимости с ошибками (Soft Margin)

На практике данные редко бывают идеально разделимы. SVM позволяет **допускать ошибки** на обучающей выборке, вводя штрафные переменные $\xi_i \ge 0$.

**Новые ограничения:**
$$d^{(i)}(w^T x^{(i)} - b_0) \ge 1 - \xi_i$$

**Целевая функция (компромисс):**
$$\frac{1}{2} w^T w + C \sum_{i=1}^{N} \xi_i \to \min$$

Здесь $C$ — гиперпараметр, управляющий компромиссом:
- **Большой $C$** → сильный штраф за ошибки → граница стремится разделить все точки (возможно переобучение)
- **Маленький $C$** → слабый штраф → широкая полоса, можно больше ошибок (возможно недообучение)

#### 2.1. Типы опорных векторов в мягком варианте

| Тип | $\lambda_i$ | $\xi_i$ | Положение |
|-----|-------------|---------|------------|
| **Периферийные** | $\lambda_i = 0$ | $\xi_i = 0$ | Далеко от границы, не влияют на решение |
| **Опорные граничные** | $0 < \lambda_i < C$ | $\xi_i = 0$ | Лежат **точно** на границе полосы |
| **Опорные нарушители** | $\lambda_i = C$ | $\xi_i > 0$ | Внутри полосы или на неправильной стороне |

### Глава 3. Нелинейный SVM (Ядерный трюк)

Если данные линейно не разделимы даже с ошибками, мы применяем **ядерный трюк** (как в лекции 9): отображаем данные в пространство **более высокой размерности** с помощью функции $\varphi(x)$, где они становятся линейно разделимыми.

**Замена скалярного произведения на ядро:**
$$x^T x^{(i)} \to K(x, x^{(i)}) = \varphi(x)^T \varphi(x^{(i)})$$

**Нелинейное решающее правило:**
$$g'(x) = \text{sign}\left( \sum_{i=1}^{N} \lambda_i d^{(i)} K(x, x^{(i)}) - b_0 \right)$$

#### 3.1. Популярные ядра для SVM

| Ядро | Формула | Применение |
|------|---------|-------------|
| **Линейное** | $K(x, z) = x^T z$ | Линейно разделимые данные |
| **Полиномиальное** | $K(x, z) = (x^T z + 1)^p$ | Нелинейные границы, $p$ — степень |
| **Радиально-базисное (RBF)** | $K(x, z) = \exp\left(-\frac{\|x - z\|^2}{2\sigma^2}\right)$ | Самый популярный, универсальный |
| **Сигмоидное** | $K(x, z) = \tanh(a x^T z + b)$ | Аналог нейросети |

#### 3.2. Преимущества RBF-ядра
- Соответствует бесконечномерному пространству
- Может аппроксимировать практически любую границу
- Имеет один параметр $\sigma$ (ширину)

### Глава 4. Мультиклассовая классификация с помощью SVM

SVM изначально решает задачу **двух классов**. Для $M > 2$ используют две стратегии:

#### 4.1. "Один против всех" (One-vs-All)
Строится $M$ бинарных классификаторов: каждый отличается свой класс от **всех остальных**.
- **Решение:** выбирается класс с максимальным значением $w_j^T x - b_{0,j}$
- **Количество классификаторов:** $M$

#### 4.2. "Попарно" (One-vs-One)
Строятся классификаторы для **каждой пары** классов $(i, j)$.
- **Решение:** голосование — каждый классификатор "отдаёт голос" за свой класс, побеждает класс с наибольшим количеством голосов
- **Количество классификаторов:** $M(M-1)/2$

На практике оба подхода работают хорошо. One-vs-One требует больше вычислений, но часто даёт лучшее качество.

### Глава 6. Достоинства и недостатки SVM

#### Достоинства
1. **Высокое качество** в широком спектре задач (один из лучших методов "из коробки")
2. **Устойчивость к переобучению** благодаря максимизации зазора
3. **Работает с большими размерностями** (ядро может уводить в бесконечномерные пространства)
4. **Решение определяется только опорными векторами** — компактное представление

#### Недостатки
1. **Чувствительность к выбросам** (особенно в линейном варианте)
2. **Трудный выбор гиперпараметра $C$ и параметров ядра ($\sigma$, $p$)** — обычно требуется кросс-валидация
3. **Медленное обучение** при больших выборках (сложность $O(N^3)$ в наивной реализации, но есть ускоренные методы)
4. **Результат плохо интерпретируем** (особенно с нелинейными ядрами)

### Итог лекции №10 (Что нужно вынести навсегда)

1. **Основная идея:** SVM ищет гиперплоскость **с максимальным зазором** между классами. Это повышает обобщающую способность.

2. **Линейный SVM (жесткий зазор):** идеально разделимые данные → задача квадратичного программирования → решение через опорные векторы.

3. **Линейный SVM (мягкий зазор):** данные с шумом → вводим штрафные переменные $\xi_i$ и параметр $C$ → компромисс между шириной полосы и ошибками.

4. **Нелинейный SVM:** данные линейно не разделимы → ядерный трюк → работаем в неявном пространстве большей размерности → нелинейная граница в исходном пространстве.

5. **Типы точек в мягком варианте:**
   - Периферийные ($\lambda_i = 0$) — не влияют
   - Опорные граничные ($0 < \lambda_i < C$) — на границе полосы
   - Опорные нарушители ($\lambda_i = C$, $\xi_i > 0$) — внутри полосы или ошибочные

6. **Главная формула (нелинейный SVM):**
   $$g'(x) = \text{sign}\left( \sum_{i \in SV} \lambda_i d^{(i)} K(x, x^{(i)}) - b_0 \right)$$

7. **Практический совет:** Начинайте с **линейного SVM** (быстро, интерпретируемо). Если качество не устраивает, пробуйте **RBF-ядро** и подбирайте $C$ и $\sigma$ по кросс-валидации.
