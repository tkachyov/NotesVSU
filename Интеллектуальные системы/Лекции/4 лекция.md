
# 4 Аппаратные аспекты реализации ИИ и работа с данными

## 4.1 Вычислительная специфика задач ИИ

Задачи ИИ обладают характерными особенностями:
- **Массовый параллелизм** вычислений
- **Преобладание операций линейной алгебры** (умножение матриц, векторов, тензоров)
- **Высокая интенсивность операций с памятью**
- **Различие требований** на этапах обучения (training) и вывода (inference)
- **Допустимость приближённых вычислений** и пониженной точности (FP16, INT8 и ниже)

Эти особенности определяют требования к архитектуре аппаратных средств.

---

## 4.2 Типы вычислительных устройств для ИИ

### 4.2.1 Центральные процессоры (CPU)
- Универсальные вычислительные устройства
- **Применение**: управление вычислениями, предобработка данных, ИИ-алгоритмы малой/средней сложности, прототипирование, отладка
- **Архитектурные особенности**: многоядерность, векторные инструкции (SSE, AVX, AVX-512), иерархия кэшей, многопоточность
- **Ограничения**: ограниченный параллелизм, низкая энергоэффективность для масштабных нейросетевых вычислений

### 4.2.2 Графические процессоры (GPU)
- **De-facto стандарт** для обучения глубоких нейронных сетей
- **Характеристики**: тысячи простых вычислительных ядер, архитектура SIMD/SIMT, высокая пропускная способность памяти, оптимизация под матричные/векторные операции
- **Применение**: обучение нейронных сетей, ускорение инференса, HPC+AI, распределённые кластеры

### 4.2.3 Специализированные ускорители

| Тип | Особенности | Применение |
|-----|-------------|-------------|
| **TPU** (Tensor Processing Unit) | Аппаратная реализация матричного умножения, высокая энергоэффективность, оптимизация под ML-фреймворки | Дата-центры, облака |
| **NPU** (Neural Processing Unit) | Высокая производительность при низком энергопотреблении | Мобильные устройства, встраиваемые системы, edge-вычисления |
| **FPGA** | Аппаратная реконфигурируемость, глубокая конвейеризация, низкие задержки | Адаптация под конкретную модель (сложность разработки, HDL) |

### 4.2.4 Типовая архитектура ИИ-ускорителя
- Вычислительные блоки (MAC, Tensor Units)
- Локальные буферы и кэш
- Контроллер памяти
- Интерфейсы высокоскоростного ввода-вывода
- Управляющая логика

> Для GPU и TPU вычислительные блоки жёстко зафиксированы, в FPGA — адаптируются.

---

## 4.3 Память и подсистема хранения данных

### 4.3.1 Типы памяти
- **DRAM** — основная память
- **HBM** (High Bandwidth Memory) — высокопропускная
- **Кэш-память**
- **Энергонезависимая память**

### 4.3.2 Узкие места
- Задержки доступа
- Пропускная способность
- Энергопотребление при передаче данных

> **Ключевой фактор**: во многих системах именно память, а не вычислительные блоки, становится основным ограничением.

---

## 4.4 Энергоэффективность и тепловые аспекты
- Современные ИИ-системы характеризуются **высоким энергопотреблением**
- **Направления оптимизации**:
  - Снижение разрядности вычислений
  - Аппаратные ускорители
  - Эффективные системы охлаждения
  - Распределение вычислений между облаком и edge-устройствами

---

## 4.5 Сравнение обучения и инференса

| Критерий | Обучение (Training) | Инференс (Inference) |
|----------|---------------------|----------------------|
| **Цель** | Оптимизация параметров | Применение обученной модели |
| **Вычислительный граф** | Forward + Backward + Update | Только Forward |
| **Основные операции** | Матричное умножение, градиенты | Матричное умножение, активации |
| **Типичные модели** | Большие и очень большие | Сжатые, оптимизированные |
| **Разрядность** | FP32, FP16, BF16 | INT8, INT4, FP16 |
| **Точность** | Критична для сходимости | Допустима аппроксимация |
| **Параллелизм** | Максимально высокий | Средний / высокий |
| **Задержки** | Вторичны | Критичны (latency-sensitive) |
| **Пропускная способность памяти** | Очень высокая | Средняя |
| **Объём памяти** | Очень большой (модель + градиенты) | Меньший (только веса) |
| **Энергопотребление** | Высокое | Низкое–умеренное |
| **Типичные платформы** | GPU, TPU, HPC-кластеры | NPU, GPU, CPU, SoC |
| **Среда выполнения** | Дата-центры, облака | Edge, мобильные, серверы |
| **Время жизни задачи** | Часы–недели | Миллисекунды–секунды |

## 4.6 Edge AI (периферийный ИИ)

### 4.6.1 Определение
Технология развертывания алгоритмов ИИ непосредственно на **локальных устройствах** (смартфоны, датчики IoT, дроны), а не в облаке.

### 4.6.2 Аппаратные требования
- Минимальное энергопотребление
- Компактность
- Работа в реальном времени
- Высокая надёжность

### 4.6.3 Примеры
- Интеллектуальные камеры
- Автономные роботы
- Системы Интернета вещей (IoT)

---

## 4.7 Работа с данными: выбор источников

При выборе источников данных необходимо учитывать:
- **Значимость** данных с точки зрения анализа
- **Сложность получения** данных из источников
- **Возможное нарушение целостности и достоверности**
- **Объём данных** в источнике

---

## 4.8 Критерии оценки качества данных

| Категория | Описание |
|-----------|----------|
| **Высокое качество** | Данные не нуждаются в очистке |
| **Некритичные ошибки** | Не мешают загрузке, но некорректны для анализа (аномалии, пропуски, дубликаты, противоречия). Могут быть исправлены в процессе анализа. |
| **Критичные ошибки** | Делают невозможной загрузку в хранилище (буква или пробел в числе, неправильный разделитель). |

---

## 4.9 Проблема множественных источников данных

### 4.9.1 Основная проблема
Выявление **перекрывающихся данных**, относящихся к одному и тому же объекту реального окружения (например, потребитель).

### 4.9.2 Триединая задача
- **Проблема идентичности объекта**
- **Проблема исключения дублирования**
- **Проблема слияния/удаления**

### 4.9.3 Принцип обработки
- **Дублирующаяся информация** → удаляется
- **Дополняющая информация** → консолидируется и соединяется
- **Цель** → согласованное представление объектов реального окружения

### 4.9.4 Пример консолидации (из лекции)
Из двух источников с разными форматами записей об одном лице (Kristen Smith / Kris L. Smith) формируется **единая консолидированная запись** с объединёнными атрибутами (адрес, телефон, пол, должность).

---

## 4.10 Заключение по аппаратной части
- Аппаратные аспекты играют **ключевую роль** в реализации ИИ
- Эффективный выбор вычислительной платформы позволяет:
  - Повысить производительность
  - Снизить энергопотребление
  - Расширить области применения ИИ-систем
- Понимание аппаратных архитектур — **необходимый элемент** подготовки специалистов
