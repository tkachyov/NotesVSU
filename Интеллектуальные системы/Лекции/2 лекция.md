
# 2 Data Science, машинное обучение и организация данных

## 2.1 Data Science (наука о данных)
- **Определение**: дисциплина, нацеленная на понимание структурированных и неструктурированных данных и поиск выводов из них.
- **Использует**: Big Data, ML, AI, глубокое обучение, Data Mining.
- **Опирается на**: анализ данных, статистику, математику, программирование, визуализацию, интерпретацию.

## 2.2 Data Mining (добыча данных)
- **Набор техник** для извлечения новой, потенциально полезной информации из больших массивов данных.
- **Концепция**: поиск скрытых паттернов и аномалий.
- **Этапы**:
  - Предварительная обработка (очистка, интеграция, преобразование)
  - Выявление паттернов
  - Представление в понятном виде
- Часть более обширной области **KDD** (Knowledge Discovery in Databases).

## 2.3 Машинное обучение (Machine Learning, ML)
- **Термин** введён в 1959 году.
- **Определение**: способность программы учиться без явного программирования.
- **Суть**: общие алгоритмы выявляют полезную информацию из данных.

### 2.3.1 Типы обучения

| Тип | Описание |
|-----|----------|
| **Supervised Learning (SL)** — обучение с учителем | Обучение на размеченных данных, указаны признаки и правильные ответы |
| **Unsupervised Learning (UL)** — без учителя | Поиск паттернов в неразмеченных данных (кластеризация) |
| **Semi-supervised learning (SSL)** | Небольшой объём размеченных + большой объём неразмеченных данных |
| **Reinforcement Learning (RL)** — с подкреплением | Обучение через пробы и ошибки, награды и наказания |
| **Self-supervised learning** | Модель предсказывает скрытую часть данных на основе открытой (автогенерация меток) |

## 2.4 Глубокое обучение (Deep Learning, DL)
- **Подмножество ML** со сложными нейронными сетями.
- Узлы (нейроны) организованы в несколько взаимосвязанных слоёв.

## 2.5 Взаимосвязь между AI, ML, DL и Data Science

```
Data Science (зонтичный термин)
  ├── Data Mining (сбор, очистка, поиск паттернов)
  ├── Machine Learning (самообучение на алгоритмах)
  │     └── Deep Learning (нейросети, многослойность)
  └── AI (функциональный продукт, решающий задачи)
```

> В большинстве проектов ИИ используется ML или DL.

# 3 MAD (Machine Learning, Artificial Intelligence & Data)

## 3.1 Что такое MAD?
- **Не дисциплина**, а зонтичное обозначение экосистемы.
- **Ключевая идея**: невозможно отделить AI, ML и Data в реальных системах.
- **Фокус**:
  - Данные как первичный актив
  - ML как основной механизм
  - AI как продуктовый и пользовательский слой
- **Не задаёт методологии** (не описывает lifecycle, не определяет как строить системы).
- **Итог**: язык управления, образования и маркетинга, а не инженерии.

## 3.2 Почему MAD появился сейчас?
1. **Размывание границ**:
   - ML без качественных данных не работает
   - AI в продакшене = ML + Data
2. **Организационный и HR-контекст**:
   - Центры компетенций (MAD teams)
   - Программы обучения, магистратуры
   - Вакансии и консалтинг
3. **Маркетинг**:
   - «AI» слишком размыто
   - «Data Science» воспринимается устаревшим
   - MAD звучит как «новая волна» (ребрендинг)

## 3.3 Сравнение Data Science, AI Engineering и MAD

| Ось сравнения | Data Science | AI Engineering | MAD |
|---------------|--------------|----------------|-----|
| Тип термина | Дисциплина | Инженерная практика | Зонтик / кластер |
| Центр внимания | Анализ и инсайты | Система и продукт | Экосистема |
| Роль данных | Источник анализа | Входной ресурс | Стратегический актив |
| Роль модели | Центральная | Компонент | Один из элементов |
| Продакшн | Вторичен | Ключевой | Подразумевается |
| Академичность | Высокая | Средняя | Низкая |
| Маркетинговость | Низкая | Средняя | Высокая |

# 4 AI Engineering

## 4.1 Определение
- **Появился как реакция** на неудачи Data Science в продакшене (≈2018 — настоящее время).
- **Ключевая идея**: AI как инженерный продукт, а не эксперимент.

## 4.2 Фокус
- Разработка end-to-end AI-систем
- Интеграция моделей в ПО
- Масштабируемость, надёжность, latency, стоимость
- MLOps / LLMOps / мониторинг / переобучение

## 4.3 Роль модели
- Модель — **один из компонентов** системы, а не центральный объект.

## 4.4 Ограничения
- Термин ориентирован прежде всего на ML/AI
- Данные подразумеваются, но не выведены в центр

## 4.5 Итог
- Инженерная дисциплина, близкая к software engineering, с AI как основным артефактом.

# 5 Слои данных в ИИ-проекте

1. **Source Systems** — источники данных
2. **Staging. Raw** — сырые данные «один в один»
3. **Staging. Intelligence** — приведение к единым единицам и названиям полей
4. **MDM (Master Data Management)** — нормативно-справочная информация (НСИ), единая запись на объект
5. **EDW (Enterprise Data Warehouse)** — хранилище с сателлитами, хабами, линками
6. **Information Delivery** — витрины данных

# 6 Типовой ИИ-проект

## 6.1 Развёртывание и мониторинг
- Модели разворачиваются на оборудовании, оптимизированном под ИИ-нагрузку (внутренние ресурсы, облако, периферия)
- Специальное аппаратное обеспечение чаще нужно для **обучения**, а не для применения
- После развёртывания — поддержка в production

## 6.2 Распределение ресурсов
- Большая часть времени и ресурсов уходит на **данные** (сбор, очистка, подготовка), а не на моделирование.

# 7 Категории AI-специалистов

| Роль | Основные задачи | Ключевые навыки |
|------|----------------|------------------|
| **Data Scientist** | Анализ данных, ML-модели, бизнес-гипотезы, отчёты | Матстатистика, Python (pandas, scikit-learn), SQL, визуализация |
| **ML Engineer** | Разработка и оптимизация моделей, интеграция в production, ML-пайплайны | ML-алгоритмы, MLOps, масштабирование |
| **Deep Learning Engineer** | Нейросети, изображения, текст, речь, эксперименты с архитектурами | PyTorch/TF, линейная алгебра, CNN/RNN/Transformers |
| **NLP Engineer** | Анализ и генерация текста, чат-боты, fine-tuning LLM | Transformers, BERT, GPT, Hugging Face, prompt engineering |
| **AI Researcher** | Исследования новых методов | Публикации, глубокое понимание теории |
| **MLOps Engineer** | Эксплуатация, мониторинг, CI/CD для моделей | DevOps + ML |
| **AI PM** | Управление AI-проектами | Понимание lifecycle AI |
| **AI Ethics** | Ответственность, безопасность, этика | Регуляторика, fairness, explainability |
