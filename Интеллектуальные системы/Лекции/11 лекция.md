# 11 Семантические сети, онтологии и логический вывод

## 11.1 Пропозициональные семантические сети

*(в лекции указан заголовок, детальное содержание — на слайдах, не попавших в текстовый слой)*

**Ключевая идея**: представление знаний в виде графа, где:
- **Узлы** — понятия (сущности, объекты, классы)
- **Рёбра** — отношения между понятиями

---

## 11.2 Концептуальные графы (Conceptual Graphs, CG)

### 11.2.1 Стандартизация
- **ISO/IEC 24707:2007** — международный стандарт для концептуальных графов
- Разработаны Джоном Соуа (John Sowa)

### 11.2.2 Основные элементы

| Элемент | Обозначение | Описание |
|---------|-------------|----------|
| **Концепты** | Прямоугольники | Объекты, атрибуты, события |
| **Концептуальные связи** | Эллипсы / стрелки | Отношения между концептами |
| **Типы** | Таксономия | Иерархия понятий (общее → частное) |

### 11.2.3 Семантика
- Базируется на логике первого порядка (FOL)
- Поддерживает:

| Операция | Описание |
|----------|----------|
| **Join** | Объединение графов по общему концепту |
| **Copy** | Копирование подграфа |
| **Restrict** | Сужение типа концепта |
| **Simplify** | Удаление избыточных связей |
| **Maximal join** | Максимальное объединение |

### 11.2.4 Преимущества
- Визуальная интуитивность
- Строгая математическая семантика
- Поддержка рассуждений (reasoning)

## 11.3 Сравнение типов семантических сетей

*(обобщение из слайда сравнения)*

| Критерий | Пропозиционная | Концептуальные графы | RDF/OWL |
|----------|----------------|---------------------|---------|
| **Семантика** | Неформальная | FOL (ISO) | DL (Description Logic) |
| **Выразительность** | Средняя | Высокая | Очень высокая |
| **Вычислимость** | Высокая | Средняя | Ограниченная (decidable) |
| **Стандартизация** | Нет | ISO | W3C |
| **Инструменты** | Кастомные | CG Tools | Protégé, RDFlib |

## 11.4 Алгоритмы обхода семантических сетей

### 11.4.1 BFS (поиск в ширину)
- **Структура данных**: очередь (FIFO)
- **Гарантирует**: кратчайший путь
- **Память**: O(ширина) — больше
- **Подходит для**: поверхностные иерархии (проверка IS-A на 2–3 уровня)

### 11.4.2 DFS (поиск в глубину)
- **Структура данных**: стек (LIFO) или рекурсия
- **Гарантирует**: не гарантирует кратчайший путь
- **Память**: O(глубина) — меньше
- **Подходит для**: глубокие иерархии (SNOMED CT: 12+ уровней)

### 11.4.3 Особенности для семантических сетей
- **Для обоих**: обязателен массив visited (предотвращение зацикливания)
- **Практический вывод**:
  - **BFS** предпочтителен при проверке достижимости (гарантирует минимальный путь)
  - **DFS** — при поиске в глубоких иерархиях и ограниченной памяти
  - **Итеративное углубление** — компромисс между памятью и оптимальностью

### 11.4.4 Псевдокоды

**BFS:**
```
function BFS(graph, start, target):
    queue = [start]
    visited = {start}
    while queue not empty:
        node = queue.pop_front()
        if node == target: return True
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append(neighbor)
    return False
```

**DFS (рекурсивный):**
```
function DFS(node, target, visited):
    if node == target: return True
    visited.add(node)
    for neighbor in graph[node]:
        if neighbor not in visited:
            if DFS(neighbor, target, visited):
                return True
    return False
```

## 11.5 Немонотонный вывод и отмена наследования

### 11.5.1 Немонотонный вывод
- **Определение**: вывод, при котором добавление нового знания может **отменить** старые заключения
- **Классический пример**: «птицы летают» + «пингвин — птица» + «пингвин не летает»

### 11.5.2 Отмена наследования (inheritance override)
- В семантических сетях свойство может быть **унаследовано** от родительского класса
- **Отмена наследования** — дочерний класс переопределяет свойство (например, отличается от родительского)
- **Механизмы**:
  - **Локальное переопределение** (explicit override)
  - **Исключение по умолчанию** (default exception)
  - **Инверсия наследования** (multiple inheritance конфликты)

### 11.5.3 Применение
- Экспертные системы (неполные или противоречивые знания)
- Обработка исключений в taxonomies
- Моделирование реальности (исключения из правил)

## 11.6 Open World Assumption (OWA) vs Closed World Assumption (CWA)

| Аспект | CWA (Закрытый мир) | OWA (Открытый мир) |
|--------|====================|====================|
| **Что не доказано** | Считается ложным | Остаётся неизвестным |
| **Типичное применение** | Реляционные БД (SQL) | Семантический веб (RDF/OWL) |
| **Пример** | В БД нет записи о клиенте → клиента не существует | Нет информации о возрасте → неизвестно, не 0 |
| **Вывод** | Полный (total) | Частичный (partial) |
| **Противоречия** | Не допускаются | Могут существовать |

### 11.6.1 Влияние на онтологии
- OWL использует **OWA** — фундаментальное отличие от реляционных БД
- Позволяет работать с **неполной информацией**
- Но усложняет вывод (отсутствие CWA означает отсутствие default-отрицания)

## 11.7 Задача → Алгоритм → Сложность

| Задача | Алгоритм | Структура | Сложность | Особенность |
|--------|----------|-----------|-----------|-------------|
| **Достижимость** | BFS | Очередь + visited | O(V + E) | Кратчайший путь |
| **Достижимость** | DFS | Стек + visited | O(V + E) | Меньше памяти |
| **Значение атрибута** | Inheritance walk | Рекурсия по IS-A | O(d × V) | Первый найденный |
| **Доказательство** | Forward chaining | Итерация (fixpoint) | O(\|R\| × \|KB\|) | Монотонный |
| **Доказательство** | Backward chaining | Дерево доказательства | P-complete | Объяснимый |
| **Вывод в OWL DL** | Reasoner (HermiT) | Tableaux | **PSPACE** | Decidable |

### 11.7.1 Практический принцип
> *Чем строже формальная семантика — тем дороже вывод. Выбирайте минимально необходимый уровень.*

## 11.8 Вероятностный вывод: Марковские логические сети (MLN)

### 11.8.1 Проблема
Реальные графы знаний (KG) содержат **неточные и противоречивые факты**

### 11.8.2 Сравнение классического и вероятностного вывода

| Аспект | Классический вывод | Марковские логические сети (MLN) |
|--------|====================|=================================|
| **Истинность правил** | 0 или 1 (абсолютно) | Правило + **вес достоверности** (0…∞) |
| **Пример правила** | «Птицы летают» (обязательно) | «Птицы летают» с весом 3.5 |
| **Противоречия** = сбой | Да | Нет (веса разрешают конфликты) |
| **Ручка исключений** | Отдельное правило «пингвины не летают» с весом 5.0 | — |
| **Сложность вывода** | NP-complete | **#P-hard** (хуже) |

### 11.8.3 Практические методы аппроксимации

| Метод | Описание |
|-------|----------|
| **MCMC** (Markov Chain Monte Carlo) | Сэмплирование из распределения по возможным мирам |
| **Belief Propagation** | Передача сообщений в факторном графе (связь с GNN) |

### 11.8.4 Применение
- Извлечение отношений из текста
- Биомедицинские графы знаний
- Рекомендательные системы

### 11.8.5 Инструменты
- **Alchemy** (UW, оригинальная реализация MLN)
- **ProbLog** (вероятностное программирование логического типа)
- **Markov Logic Engine** (MLN)

## 11.10 Онтологии: определение

### 11.10.1 Философское определение
> *Учение о бытии и сущностях мира* (Аристотель, XVII–XX вв.)

### 11.10.2 Определение в информатике
> *Формальная спецификация концептуализации предметной области*

### 11.10.3 Инженерное определение
> *Артефакт ИИ-системы: разделяемый, повторно используемый словарь знаний*

### 11.10.4 Определение Грубера (1993) — классическое
> **«Онтология — это формальная явная спецификация разделяемой концептуализации»**

**Расшифровка**:
- **Формальная** — машинно-читаемая
- **Явная** — все понятия и ограничения заданы явно
- **Разделяемая** — общая для сообщества
- **Концептуализация** — абстрактная модель предметной области

## 11.11 Формальная модель онтологии

### 11.11.1 Базовая модель (С)

**O = ⟨C, P, R, A⟩**

где:
- **C** — конечное множество понятий (классов сущностей)
- **P** — конечное множество свойств этих понятий
- **R** — конечное множество связей между понятиями
- **A** — множество аксиом (утверждений, ограничений)

### 11.11.2 Расширенная модель (с лексикой)

**O = ⟨C, P, R, A, L, F, G, H⟩**

где дополнительно:
- **L** = L<sub>C</sub> ∪ L<sub>R</sub> — словарь (лексические единицы для понятий и отношений)
- **F, G** — функции ссылок (лексическая единица → понятие/отношение; одна единица может ссылаться на несколько понятий, и наоборот)
- **H** ⊆ C × C — таксономическая иерархия (нерефлексивная, ациклическая, транзитивная):
  - H(C₁, C₂) означает, что понятие C₁ является **подпонятием** (subclass) C₂

## 11.12 Спектр онтологий: от словаря к OWL

### 11.12.1 Иерархия уровней

| Уровень | Тип | Описание |
|---------|-----|----------|
| **Словарь** (Glossary) | Минимальный | Простой список терминов и их определений |
| **Таксономия** (Taxonomy) | IS-A | Иерархия понятий (общее → частное) |
| **Теззаурус** (Thesaurus) | Связи | Таксономия + ассоциативные отношения (BT, NT, RT, USE) |
| **Неформальная онтология** | Ограничения | + доменные и ранговые ограничения |
| **Формальная онтология** | Логика | + аксиомы, непротиворечивость, логический вывод |
| **OWL-онтология** | Max | Формальная семантика (Description Logic), автоматический вывод (reasoner), проверка согласованности |

### 11.12.2 Ключевые тезисы
- **OWL-онтология** — наивысший уровень (формальная семантика, автоматический вывод, верификация)
- **Все нижние уровни** — её частные случаи или предшественники

## 11.13 Сравнение CWA vs OWA в контексте онтологий

| Аспект | CWA (Базы данных) | OWA (Онтологии OWL) |
|--------|===================|=====================|
| **Значение отсутствующего факта** | Ложь (false) | Неизвестно (unknown) |
| **Противоворечия** | Не допускаются | Могут существовать |
| **Вывод** | Полный (замыкание) | Частичный (открытый) |
| **Пример** | SELECT ... WHERE | rdf:type + owl:sameAs |
| **«Человек с неизвестным возрастом»** | Возраст = NULL (не 0) | Возраст неизвестен ⇒ не выводится |

## 11.14 Итоговая таблица логических подходов

| Подход | Формализм | Сложность | Масштаб | Объяснимость | Применение |
|--------|===========|===========|=========|=============|============|
| **Классический FOL** | ∃, ∀, →, ∧, ∨ | NP | Малый | Высокая | Эксп. системы |
| **Forward chaining** | Итерация по фактам | Линейная* | Средний | Средняя | Базы правил |
| **Backward chaining** | Дерево доказательства | P-complete | Малый | Высокая | Prolog |
| **OWL reasoning** | Description Logic (SHOIN/DL) | PSPACE | Средний | Средняя | Семантический веб |
| **MLN (Markov Logic)** | FOL + веса | #P-hard | Средний | Низкая | KG, извл. текста |

*зависит от числа фактов и правил (итерация до фиксации)
