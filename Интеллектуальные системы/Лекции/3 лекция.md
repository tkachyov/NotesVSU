
# 3 Данные, знания и их свойства в ИИ

## 3.1 Данные и знания: базовые определения
- **Данные** — отдельные факты, характеризующие объекты, процессы и явления в предметной области, а также их свойства.
- **Знания** — выявленные закономерности предметной области (принципы, связи, законы), позволяющие решать задачи.

---

## 3.2 Свойства знаний

### 3.2.1 Внутренняя интерпретируемость
- Каждая информационная единица (концепт, факт, правило, объект) имеет уникальный идентификатор в пределах системы.
- **Инженерный смысл**: устраняется неоднозначность ссылок, обеспечивается адресуемость знаний.
- **Примеры реализации**: URI в онтологиях (OWL/RDF), уникальные ключи в БД, идентификаторы узлов в графах знаний.

### 3.2.2 Структурированность
- Знания организованы в формальные структуры (иерархии, графы, фреймы, правила).
- **Формы структурирования**: таксономии (is-a), мерономии (part-of), продукционные правила, семантические сети, онтологии.

### 3.2.3 Связность
- Между элементами знаний существуют явно заданные отношения.
- **Типы связей**: иерархические, ассоциативные, причинные, временные, функциональные.

### 3.2.4 Семантическая метрика
- Механизм оценки близости или релевантности знаний на основе смысла (не сравнение строк).
- **Примеры**: расстояние в онтологическом графе, косинусная близость в embeddings, меры Resnik, Lin.

### 3.2.5 Активность
- Знания способны инициировать выводы или действия (продукционные системы IF–THEN, триггеры, логический вывод).

### 3.2.6 Непротиворечивость
- В системе знаний отсутствуют логические конфликты (нет одновременного утверждения P и ¬P).
- **Инструменты**: логические резонеры, SAT/SMT-решатели, онтологическая проверка консистентности.

---

## 3.3 Требования к обнаруживаемым знаниям
- **Новыми** — иначе усилия не окупаются.
- **Нетривиальными** — отражать неочевидные, неожиданные закономерности.
- **Практически полезными**.
- **Доступными для понимания человеком** — иначе риск случайности.

---

## 3.4 Данные как «топливо» для ИИ
- **Обучающая выборка (датасет)** — массивы для тренировки нейросетей.
- **Извлечение признаков** — выделение ключевых атрибутов (формы, частота слов).
- **Масштабируемость**: чем качественнее и объёмнее данные, тем точнее data-driven модели (GPT, распознавание лиц).

## 3.5 Знания как интерпретация и контекст
- **Базы знаний** — хранят не просто факты, а структурированные правила и логические выводы (онтологии).
- **Экспертные системы** — классический ИИ на правилах «если–то» (прозрачные, объяснимые).
- **Принятие решений** — знание позволяет действовать осознанно в узких областях (медицина).

## 3.6 Модель DIKW (Data–Information–Knowledge–Wisdom)

| Уровень | Описание |
|---------|----------|
| **Данные (Data)** | Дискретные объективные факты (например, «38.5») |
| **Информация (Information)** | Данные с контекстом (кто, что, где) |
| **Знания (Knowledge)** | Прикладная информация, понимание как использовать данные |
| **Мудрость (Wisdom)** | Понимание причинно-следственных связей и этический контекст |

### 3.6.1 Сопоставление свойств знаний с уровнями DIKW

| Свойство | Data | Information | Knowledge | Wisdom | Пояснение |
|----------|------|-------------|-----------|--------|------------|
| Внутренняя интерпретируемость | часто | ✓ | ✓ | ✓ | Уникальная идентификация начинается с информации |
| Структурированность | × | ✓ | ✓ | ✓ | Структура превращает данные в информацию |
| Связность | × | часто | ✓ | ✓ | Связи формируют знание |
| Семантическая метрика | × | часто | ✓ | ✓ | Смысл появляется с уровня знания |
| Активность | × | × | ✓ | ✓ | Знание инициирует вывод, мудрость — действия |
| Непротиворечивость | не критично | желательно | ✓ | ✓ | Консистентность обязательна для знаний |

## 3.7 Базовая онтологическая модель данных

**D = (S, M, R, G, T, L, P, Q)**

| Компонент | Описание |
|-----------|----------|
| **S (Structure)** | синтаксическая организация |
| **M (Modality)** | модальность данных |
| **R (Role)** | функциональная роль в обучении |
| **G (Genesis)** | происхождение |
| **T (Temporal nature)** | временная природа |
| **L (Labeling regime)** | режим разметки |
| **P (Privacy/Policy class)** | правовой статус |
| **Q (Quality/Distributional properties)** | распределительные и качественные характеристики |

### 3.7.1 Измерение S — структурная организация
- **S1. Структурированные**: табличные (relational), time-indexed tabular
- **S2. Полуструктурированные**: JSON / XML, логи
- **S3. Неструктурированные**: текст, изображения, аудио, видео
- **S4. Графовые**: социальные графы, knowledge graphs

### 3.7.2 Измерение M — модальность
- M1. Текстовая (LLM, NLP)
- M2. Визуальная (компьютерное зрение)
- M3. Аудиальная (ASR, TTS)
- M4. Видео (spatiotemporal)
- M5. Графовая (соцсети, графы знаний)
- M6. Табличная (градиентный бустинг)
- M7. Мультимодальная (изображение + текст)

## 3.8 Сравнение данных и знаний в ИИ

| Характеристика | Данные (Data-driven AI) | Знания (Knowledge-based AI) |
|----------------|------------------------|-----------------------------|
| Источник | Сырые массивы, Big Data, сигналы | Экспертный опыт, правила, логика |
| Механизм | Статистическое обучение, нейросети | Логический вывод, поиск в графах |
| Хранение | Базы данных (SQL/NoSQL) | Базы знаний (онтологии, графы) |
| Извлечение | Data Mining, ML | Инженерия знаний, эксперты |
| Сильные стороны | Гибкость, работа с неструктур. контентом | Точность, объяснимость, стабильность |
| Слабые стороны | «Чёрный ящик», огромные ресурсы | Сложность обновления, ограниченность |
| Пример в ИИ | Веса в нейронной сети | Правило «Если A, то B» |

## 3.9 Типы данных по структурированности

| Характеристика | Структурированные | Неструктурированные | Полуструктурированные |
|----------------|-------------------|---------------------|------------------------|
| Форматы | Таблицы, строки, столбцы | Текст, аудио, видео, изображения | XML, JSON, CSV |
| Модель данных | Реляционная | None | Иерархическая / Графовая |
| Хранилища | Реляционные СУБД, DWH | File systems, Data Lakes, облако | NoSQL БД |
| Природа данных | Чётко определены, фикс. схема | Непредсказуемы, схема отсутствует | Свободная схема |
| Анализ | SQL, Data Mining | NLP, распознавание образов | Query languages, Data Mining |
| Инструменты | MS SQL, Oracle, MySQL | S3, Hadoop, Spark | MongoDB, Cassandra, Couchbase |

### 3.9.1 Неструктурированные данные: категории
- **Генерируемые людьми**: текстовые документы, email, посты в соцсетях, изображения, видео
- **Машинные**: файлы журналов (логи), данные GPS, IoT, телеметрия

## 3.10 Разметка данных и мета-данные

### 3.10.1 Разметка (labeling)
- Процесс идентификации необработанных данных и добавления меток для обучения с учителем.
- Для повышения эффективности — автоматическая маркировка (модель обучается на подмножестве, размеченном человеком).

### 3.10.2 Мета-данные (metadata)
- **Данные о данных** — структурированная информация, описывающая свойства, происхождение, контекст, назначение.
- **Данные** = контент (текст, изображение), **мета-данные** = что, когда, кем, в каком формате, с каким смыслом.

### 3.10.3 История развития мета-данных
- **1968** — MARC (машинная обработка библиотечных записей)
- **1986** — SGML (мета-языки, теги, атрибуты)
- **1995** — Dublin Core (15 элементов: название, автор, тема, язык, формат и др.)
- **1999-2004** — RDF, OWL (семантический веб, связи между объектами)

### 3.10.4 Типы мета-данных
- **Описательные** — автор, дата, тема, аннотация (каталоги, поиск)
- **Структурные** — формат, поля, связи, последовательность (главы книги, кадры видео)
- **Административные** — лицензии, права доступа, история модификаций
- **Технические** — разрешение, формат файла, кодировка, объём
- **Семантические** — смысловые связи, термины, концепции (онтологии, ИИ)

### 3.10.5 Data Lineage (происхождение данных)
- Мета-данные фиксируют путь каждой единицы информации: источник, трансформации, фильтрацию, валидацию.
- Без мета-данных — «чёрный ящик». Мета-данные возвращают прозрачность.

### 3.10.6 Управление версиями данных
- **Инструменты**: DVC (Data Version Control), MLflow
- Позволяют воспроизвести обучение модели, проверить корректность экспериментов.

### 3.10.7 Пример разметки (Object Detection)
```json
{"image_id": 101, "category": "car", "bbox": [120, 80, 200, 130]}
```

## 3.11 Синтетические данные (Synthetic Data)

### 3.11.1 Определение
- Информация, созданная искусственно алгоритмами или симуляциями, а не собранная в реальных событиях.
- Имитирует статистические свойства реальных данных, но без конфиденциальной информации.

### 3.11.2 Ключевые особенности
- **Искусственное происхождение** (GAN, VAE, Diffusion models, скрипты)
- **Защита конфиденциальности** (безопасно передавать без нарушения GDPR, 152-ФЗ)
- **Контроль параметров** (редкие сценарии, балансировка датасетов)

### 3.11.3 Области применения
- **ML** — когда реальных данных мало или они дороги (медицинские снимки редких болезней)
- **Тестирование ПО** — без риска утечки реальной информации
- **Финансы** — симуляция рынка, обучение систем обнаружения мошенничества
- **Автономный транспорт** — виртуальные среды для тренировки беспилотников

### 3.11.4 Сравнение реальных и синтетических данных

| Характеристика | Реальные данные | Синтетические данные |
|----------------|-----------------|----------------------|
| Достоверность | Отражают реальный мир | Могут упускать скрытые закономерности |
| Приватность | Высокий риск утечки | Практически полная безопасность |
| Стоимость | Дорого и долго | Быстрая и дешёвая генерация |
| Масштабируемость | Ограничены физическим миром | Неограниченны |

### 3.11.5 Способы генерации
1. **Статистическое моделирование**: многомерные распределения, байесовские сети
2. **Генеративные модели глубокого обучения**: GAN, VAE, Diffusion models
3. **Симуляция и процедурная генерация**: физические движки, игровые движки, CAD/CAE, агентные модели

### 3.11.6 Главный риск
- Модель может **переучиться** на искусственных паттернах и плохо работать в реальном мире, если генератор был недостаточно точным.

## 3.12 Вычислительная специфика задач ИИ
- Массовый параллелизм вычислений
- Преобладание операций линейной алгебры (умножение матриц, векторов, тензоров)
- Высокая интенсивность операций с памятью
- Различие требований на этапах **обучения (training)** и **вывода (inference)**
- Допустимость приближённых вычислений и пониженной точности (FP16, INT8 и ниже)
