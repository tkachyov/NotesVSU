# 9 BI-инструменты, визуализация и AI в аналитике

## 9.1 Критерии выбора BI-инструмента

*(в лекции указан заголовок, содержание — на слайдах, не попавших в текстовый слой)*

Основные критерии (обобщение):
- Поддержка источников данных (OLAP, Data Lake, Lakehouse)
- Наличие AI-функций (NLQ, объяснение результатов, прогнозирование)
- Self-service возможности для бизнес-пользователей
- Масштабируемость и производительность
- Интеграция с семантическим слоем

## 9.2 VizQL: визуальный язык запросов

### 9.2.1 Концепция
**Drag & Drop → VizQL → SQL/MDX → Chart**

- Пользователь перетаскивает поля мышью
- VizQL (Visual Query Language) автоматически генерирует запрос
- Запрос выполняется к источнику (SQL для ROLAP, MDX для MOLAP)
- Результат — визуализация

### 9.2.2 Значение для AI
- Снижает порог входа для бизнес-аналитики
- Автоматизация построения графиков
- Основа для **NLQ** (Natural Language Query) — пользователь пишет вопрос, система переводит в VizQL/SQL

## 9.3 Tableau и AI-функции

### 9.3.1 Tableau Pulse
- Автоматическое обнаружение аномалий и изменений в метриках
- Персонализированные дайджесты изменений
- Объяснение причин изменений (drive-by analysis)

### 9.3.2 Tableau GPT
- NLQ на естественном языке
- Автоматическое создание дашбордов по описанию
- Генерация пояснений к данным

### 9.3.3 Другие AI-функции Tableau
| Функция | Назначение |
|---------|------------|
| **Explain Data** | Автоматический поиск объяснений аномалии |
| **Ask Data** | Вопрос на естественном языке → визуализация |

## 9.4 Power BI: DAX и семантическая модель

### 9.4.1 DAX (Data Analysis Expressions)
- Язык формул для Power BI
- Создание **вычисляемых колонок** и **мер**
- Учитывает **контекст** (фильтры, строки, визуализации)

### 9.4.2 Типы вычислений в DAX

| Тип | Описание | Пример |
|-----|----------|--------|
| **Вычисляемая колонка** | Строка за строкой, вычисляется при загрузке | `[Price] * [Quantity]` |
| **Мера** | Агрегация с учётом контекста фильтров | `SUM(Sales[Amount])` |

### 9.4.3 Семантическая модель в Power BI
- Набор таблиц, связей, мер и иерархий
- Единое бизнес-определение метрик
- Основа для **self-service аналитики**

### 9.4.4 Power BI Copilot
- **GPT-4 внутри Microsoft 365**
- Создание отчётов и дашбордов по текстовому описанию
- NLQ на естественном языке
- Автоматическая генерация DAX-формул
- Объяснение существующих визуализаций

## 9.5 Принципы дизайна дашборда

### 9.5.1 Ключевая установка
> *Аналитический дашборд — инструмент решений, не «красивый отчёт»*

### 9.5.2 Основные принципы

| Принцип | Описание |
|---------|----------|
| **Целеполагание** | Каждая метрика должна отвечать на бизнес-вопрос |
| **Иерархия информации** | Главное на видном месте, детали — под капотом |
| **Консистентность** | Единые цвета, шрифты, типы визуализаций |
| **Минимализм** | Не перегружать — убрать лишнее |
| **Actionability** | Дашборд должен подсказывать, что делать |
| **Контекст** | Сравнение с планом, прошлым периодом, бенчмарком |

### 9.5.3 Типичные ошибки
- Слишком много визуализаций на одном экране
- Отсутствие пояснений (что означает метрика)
- Использование неподходящих типов графиков
- Игнорирование мобильной версии

---

## 9.6 Типы дашбордов

| Тип | Целевая аудитория | Характеристики | Примеры метрик |
|-----|-------------------|----------------|----------------|
| **Операционный** | Линейные менеджеры, операторы | Обновление в реальном времени/часы, детальный уровень | Количество заказов сегодня, загрузка склада |
| **Стратегический** | Топ-менеджмент | Обновление ежемесячно/ежеквартально, агрегированные KPI | Выручка по рынкам, доля рынка, ROI |
| **Аналитический** | Аналитики, дата-сайентисты | Глубокое исследование, срезы, фильтры | Когортный анализ, RFM-сегментация |

## 9.7 Self-Service аналитика: паттерны

### 9.7.1 Три архетипа организации

| Паттерн | Кто создаёт отчёты | Плюсы | Минусы |
|---------|-------------------|-------|--------|
| **Централизованный** | Специализированная BI-команда | Контроль качества, единые стандарты | Узкое горлышко, долгие сроки |
| **Децентрализованный** (Self-Service) | Бизнес-пользователи | Быстрота, гибкость | Разнобой в метриках, недоверие к данным |
| **Managed Self-Service** | Бизнес-пользователи + семантический слой + governance | Скорость + качество + единые определения | Требует зрелости организации |

### 9.7.2 Тренд 2025
> *Managed Self-Service + Semantic Layer = оптимальный баланс*

**Ключевые элементы**:
- **Семантический слой** (dbt, Cube) — единое определение метрик
- **Data Governance** — политики качества и доступа
- **Self-service BI** (Power BI, Tableau) — интерфейс для бизнеса
- **Центр компетенций** — поддержка и обучение

## 9.8 Сравнение Tableau и Power BI (дополнение)

| Критерий | Tableau | Power BI |
|----------|---------|----------|
| **Визуализация** | Сильнейшая (VizQL) | Хорошая, но уступает Tableau |
| **Интеграция с Microsoft** | Средняя | Отличная (Excel, Teams, Azure) |
| **Язык вычислений** | Tableau Calculations | DAX |
| **AI-функции** | Tableau GPT, Pulse | Copilot (GPT-4) |
| **Self-service** | Очень сильный | Очень сильный |
| **Цена** | Выше | Ниже (в экосистеме MS) |
| **Семантический слой** | Tableau Data Model | Power BI Dataset |
| **Облачная платформа** | Tableau Cloud | Power BI Service |

## 9.9 Роль AI в современных BI-инструментах

### 9.9.1 Ключевые AI-возможности

| Возможность | Описание | Примеры |
|-------------|----------|---------|
| **NLQ** (Natural Language Query) | Вопрос на естественном языке → визуализация | Ask Data (Tableau), Copilot (Power BI) |
| **NLG** (Natural Language Generation) | Автоматическое текстовое описание данных | Tableau Pulse |
| **Автоматическое объяснение** | Поиск причин аномалии | Explain Data (Tableau) |
| **Прогнозирование** | Встроенные временные ряды | Forecast в Tableau/Power BI |
| **Обнаружение аномалий** | Автоматический мониторинг метрик | Tableau Pulse |
| **Генерация отчётов** | Создание дашборда по тексту | Copilot (Power BI) |
| **Кластеризация** | Автоматическая сегментация | Clustering в Tableau |


## 9.10 Ключевые термины лекции

| Термин | Определение |
|--------|-------------|
| **BI** | Business Intelligence — бизнес-аналитика |
| **VizQL** | Visual Query Language — язык визуальных запросов (Tableau) |
| **DAX** | Data Analysis Expressions — язык формул Power BI |
| **Вычисляемая колонка** | DAX-формула, вычисляемая при загрузке данных |
| **Мера** | DAX-формула с агрегацией, вычисляемая в контексте фильтров |
| **NLQ** | Natural Language Query — запрос на естественном языке |
| **NLG** | Natural Language Generation — генерация текстовых описаний |
| **Semantic Layer** | Слой единых определений метрик |
| **Managed Self-Service** | Самообслуживание с управлением и семантическим слоем |
| **Copilot** | AI-ассистент Power BI на базе GPT-4 |
| **Tableau Pulse** | AI-мониторинг аномалий и изменений метрик |
