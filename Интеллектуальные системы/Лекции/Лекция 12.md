# 12 Онтологии: определение, спектр, применение и ограничения

## 12.1 Что такое онтология?

### 12.1.1 Определения в разных контекстах

| Контекст | Определение |
|----------|-------------|
| **Философия** | Учение о бытии и сущностях мира (Аристотель, XVII–XX вв.) |
| **Информатика** | Формальная спецификация концептуализации предметной области |
| **Инженерный (ИИ)** | Артефакт ИИ-системы: разделяемый, повторно используемый словарь знаний |

### 12.1.2 Классическое определение Грубера (1993)

> **«Онтология — это формальная явная спецификация разделяемой концептуализации»**

**Расшифровка терминов**:

| Термин | Значение |
|--------|----------|
| **Формальная** | Машинно-читаемая, строгая семантика |
| **Явная** | Все понятия и ограничения заданы явно (не неявно, не процедурно) |
| **Разделяемая** | Общая для сообщества (не персональная) |
| **Концептуализация** | Абстрактная модель предметной области |

## 12.2 Спектр онтологий: от словаря к OWL

*(Лекция 11 уже содержала эту таблицу, здесь она повторяется в контексте лекции 12)*

| Уровень | Тип | Описание |
|---------|-----|----------|
| **Словарь** (Glossary) | Минимальный | Простой список терминов и определений |
| **Таксономия** (Taxonomy) | IS-A | Иерархия понятий (общее → частное) |

| **OWL-онтология** | Максимальный | Формальная семантика (Description Logic), reasoner |

## 12.3 Онтологии верхнего уровня (Upper-Level Ontologies)

### 12.3.1 Назначение
При разработке **мультиагентных систем** агенты из разных предметных областей должны «понимать» друг друга. Онтологии верхнего уровня служат для **семантической интеграции**:
- Агент системы видеонаблюдения и агент финансовой системы оба знают, что такое «Организация» и «Событие» — через SUMO или DOLCE

### 12.3.2 Сравнение основных онтологий верхнего уровня

| Характеристика | SUMO | DOLCE | Онтология Совы (Sowa) |
|----------------|------|-------|----------------------|
| **Авторы** | IEEE / Teknowledge | Lab. for Applied Ontology (Trento) | John Sowa |
| **Основа** | Теория множеств + KIF-аксиомы | Когнитивная лингвистика + философия | Логика + философия + лингвистика |
| **Концептов** | ~1 000 | ~200 базовых категорий | Каркас различий (12 категорий) |
| **Аксиом** | ~4 000 | — | — |
| **Ключевая идея** | Строгая иерархия, MILO-расширение | Endurant / Perdurant разделение | «Бриллиант»: 4 пары различий |
| **Связь с NLP** | Отображение на WordNet | Лингво-когнитивная инженерия | Концептуальные графы |
| **Лицензия** | Открытая (BSD-like) | Открытая | Открытая (книги, статьи) |

### 12.3.3 Что такое Endurant и Perdurant?

| Тип | Описание | Пример |
|-----|----------|--------|
| **Endurant** (континуант) | Сущность, существующая целиком в каждый момент времени | Человек, машина, город |
| **Perdurant** (оккурент) | Сущность, разворачивающаяся во времени (событие, процесс) | Полёт, переговоры, землетрясение |

## 12.4 Структура онтологий: свойства

*(Лекция содержит заголовки «Свойства: объектные и свойства данных», но детали — на слайдах без текстового слоя)*

### 12.4.1 Объектные свойства (Object Properties)
- Связывают **объекты с объектами**
- Пример: `isDirectedBy` (Фильм → Режиссёр)
- В OWL: `ObjectProperty`

### 12.4.2 Свойства данных (Data Properties)
- Связывают **объекты с литералами** (числа, строки, даты)
- Пример: `hasRating` (Фильм → 9.3)
- В OWL: `DataProperty`

### 12.4.3 Характеристики свойств

| Характеристика | Значение |
|----------------|----------|
| **Functional** | Одно значение для каждого объекта |
| **InverseFunctional** | Уникальный объект для каждого значения |
| **Transitive** | Если a R b и b R c, то a R c |
| **Symmetric** | Если a R b, то b R a |
| **Asymmetric** | Если a R b, то не b R a |
| **Reflexive** | a R a всегда |
| **Irreflexive** | a R a никогда |

## 12.5 Аксиомы и ограничения

*(Лекция содержит заголовок, детали — на слайдах)*

**Аксиомы** в онтологиях задают:
- **Непротиворечивость** (например, человек не может быть одновременно младше 0 лет)
- **Диапазоны свойств** (размер: 0–1000)
- **Кардинальность** (у студента ровно 1 ID)
- **Дизъюнктные классы** (Мужчина и Женщина не пересекаются)
- **Эквивалентность классов и свойств**

**Пример аксиомы в OWL**:
```owl
:Student ⊑ :Person ⊓ ∃:hasID exactly 1 :StudentID
```

## 12.6 Нейросимвольная интеграция

### 12.6.1 Концепция
Символьный вывод (онтологии, логика) + нейросетевое обучение (статистика, эмбеддинги) в одной системе.

### 12.6.2 Типы интеграции

| Тип | Описание | Пример |
|-----|----------|--------|
| **Символьный слой над нейросетью** | Нейросеть извлекает факты, онтология проверяет / выводит | Извлечение отношений из текста → OWL reasoner |
| **Нейросеть с онтологическими ограничениями** | Ontology-aware обучение (потери учитывают аксиомы) | Онтологические потери (βάρος, англ. weight) в embedding space |
| **Эмбеддинги онтологий** | Узлы KG → векторы, поддержка вывода через ANN | RDF2Vec, OWL2Vec*, ComplEx |
| **LLM + KG (GraphRAG)** | LLM генерирует запросы к графу знаний | RAG с Neo4j вместо векторной БД |

### 12.6.3 Преимущества гибрида

| Онтология даёт | Нейросеть даёт |
|----------------|----------------|
| Логическая непротиворечивость | Обобщение от примеров |
| Объяснимость (трассировка вывода) | Работа с неструктурированным контентом (текст, изображения) |
| Контроль над предметной областью | Масштабирование за счёт векторных представлений |
| Формальная верификация | Устойчивость к шуму и неполноте |

## 12.7 Ограничения и сфера применимости онтологий

### 12.7.1 Основные ограничения

| Ограничение | Описание |
|-------------|----------|
| **Высокая стоимость создания** | Разработка качественной онтологии требует экспертизы в предметной области и многих месяцев работы. Автоматизация лишь частична. |
| **Сложность поддержки** | Эволюция предметной области требует обновления онтологии, что может нарушить согласованность всей базы знаний. |
| **OWA мешает ML** | Алгоритмы машинного обучения предполагают CWA. Гибридизация требует явного согласования двух предположений о мире. |
| **Масштабирование вывода** | Полный вывод на OWL 2 DL — сложность **NExpTime**. Для промышленных графов используют подмножества (EL, QL, RL). |
| **Порог вхождения** | Логика дескрипций, SPARQL, Protégé — высокая кривая обучения. Сопротивление в командах без специалистов. |

### 12.7.2 Когда онтологии НЕ нужны

> *Если данные плоские и хорошо структурированы (реляционная БД) — онтология избыточна. Оценивайте соотношение выгода / сложность.*

### 12.7.3 Когда онтологии нужны

- **Сложные предметные области** с множеством связей и правил
- **Интеграция гетерогенных источников** (разные схемы БД, JSON, XML)
- **Семантический веб** / Linked Data
- **Мультиагентные системы** с необходимостью общей семантики
- **Агенты с объяснимым выводом** (например, медицинские экспертные системы)
- **Контроль знаний** (нормативные документы, законодательство)

### 12.7.4 Практические профили OWL для масштабирования

| Профиль | Сложность | Применение |
|---------|-----------|------------|
| **OWL EL** | Полиномиальная | Биомедицина, таксономии с большим числом классов |
| **OWL QL** | AC0 (SQL-сводимый) | Онтологии как оболочка над реляционными БД |
| **OWL RL** | Полиномиальная | Правила продукций (rule-based reasoning) |

## 12.8 Итоговая таблица: онтологии в ИИ

| Аспект | Описание |
|--------|----------|
| **Основное назначение** | Формальная спецификация концептуализации, разделяемая и повторно используемая |
| **Базовые элементы** | Классы (C), свойства (P), отношения (R), аксиомы (A) |
| **Язык** | OWL (W3C), базируется на дескрипционных логиках |
| **Предположение о мире** | OWA (Open World Assumption) |
| **Вывод** | Reasoner (HermiT, Pellet, Fact++) — сложность от PSpace до NExpTime |
| **Ключевые ограничения** | Высокая стоимость создания, сложная поддержка, масштабирование |
| **Гибридизация** | Нейросимвольная интеграция (LLM + KG, ontologically-informed ML) |
| **Когда применять** | Сложные, гетерогенные домены с потребностью в выводе и интеграции |
