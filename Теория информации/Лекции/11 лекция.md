# Конспект лекции №11: Арифметическое кодирование и введение в помехоустойчивое кодирование

## 1. Оценка избыточности кода Хаффмена (продолжение)

### 1.1. Теорема Галлагера (теорема 3.4)
Пусть $p_1$ — наибольшая вероятность сообщения. Тогда избыточность $r = \bar{\ell} - H$ кода Хаффмена удовлетворяет:

$$
r \le \begin{cases}
p_1 + \sigma, & \text{если } p_1 < 1/2, \\[4pt]
2 - \eta(p_1) - p_1, & \text{если } p_1 \ge 1/2,
\end{cases}
$$

где
- $\eta(x) = -x\log x - (1-x)\log(1-x)$ — энтропия двоичного ансамбля,
- $\sigma = 1 - \log e - \log \log e \approx 0.08607$.

**Значение:** Оценка довольно точна для широкого диапазона $p_1$, но при $p_1$, близком к 1, может быть завышена.

## 2. Код Шеннона (ещё раз)

### 2.1. Построение
1. Упорядочить буквы по убыванию вероятностей: $p_1 \ge p_2 \ge \dots \ge p_M$.
2. Вычислить кумулятивные вероятности:
   $$q_1 = 0,\quad q_2 = p_1,\quad q_3 = p_1+p_2,\quad \dots$$
3. Длина кодового слова для $m$-й буквы: $\ell_m = \lceil -\log p_m \rceil$.
4. Кодовое слово — первые $\ell_m$ битов двоичного представления $q_m$.

### 2.2. Свойства
- Средняя длина $\bar{\ell} < H + 1$.
- Код не оптимален (хуже кода Хаффмена), но проще в построении.
- Является прообразом арифметического кодирования.

---

## 3. Переход к блочному (FV) кодированию

### 3.1. Мотивация
- **Учёт памяти источника:** Побуквенные методы не используют зависимость между символами.
- **Проблема малой энтропии:** При $H < 1$ побуквенное кодирование даёт $\bar{\ell} \ge 1$, что далеко от $H$.
- **Решение:** Кодировать **блоки** длины $n$ (FV-кодирование).

### 3.2. Определения
- Средняя скорость кодирования для блоков длины $n$:
  $$\overline{R}_n = \frac{1}{n} M[\ell(x)].$$
- Средняя скорость (предельная):
  $$\overline{R} = \inf_n \overline{R}_n.$$

---

## 4. Теоремы для FV-кодирования стационарного источника

### 4.1. Обратная теорема (теорема 3.5)
Для стационарного источника с энтропией $H$:
$$R \ge H.$$

**Доказательство:**
$$M[\ell(x)] \ge H(X^n) = n H_n(X) \ge n H_\infty(X) = n H.$$

### 4.2. Прямая теорема (теорема 3.6)
Для любого $\delta > 0$ существует FV-код такой, что:
$$R \le H + \delta.$$

**Доказательство (основные шаги):**
1. Прямая теорема для побуквенного кодирования: $M[\ell(x)] \le H(X^n) + 1 = n H_n(X) + 1$.
2. Выбираем $n$ достаточно большим, чтобы $|H_n(X) - H| \le \delta/2$ и $1/n \le \delta/2$.
3. Тогда:
   $$\overline{R}_n \le H_n(X) + \frac{1}{n} \le H + \delta.$$

### 4.3. Теоретический итог
$$H \le \overline{R} \le H + o(n), \quad o(n) \to 0 \text{ при } n \to \infty.$$

---

## 5. Проблема практической реализации

- При блочном кодировании алфавит "расширенного" источника $X^n$ имеет размер $|X|^n$, что экспоненциально растёт с $n$.
- Даже при $|X|=256$ и $n=4$ объём алфавита достигает $256^4 \approx 4.3 \cdot 10^9$.
- **Вывод:** Прямое использование побуквенных кодов для блоков невозможно из-за экспоненциальной сложности.

## 6. Арифметическое кодирование

### 6.1. Идея метода
- Обобщение кода Шеннона на последовательности.
- Не требует явного перечисления всех блоков.
- Каждой последовательности ставится в соответствие **интервал** $[F, F+G)$ на $[0,1)$, где $F$ — начальная точка, $G$ — длина интервала.
- Кодом служит двоичное представление любой точки внутри интервала (обычно $F$ с округлением).

### 6.2. Алгоритм (для источника без памяти)
1. Исходный интервал $[0,1)$.
2. Для каждого символа последовательности интервал сужается до подынтервала, соответствующего этому символу (длина подынтервала пропорциональна вероятности символа).
3. После обработки всей последовательности интервал $[F, F+G)$ заменяется его двоичным представлением.

### 6.3. Сложность
- Наивная реализация требует работы с числами растущей точности: после $n$ шагов точность $\approx nd$ бит.
- Суммарная сложность: $\Theta(n^2)$.
- Однако существуют методы, позволяющие снизить сложность до $\Theta(n)$ с пренебрежимо малыми потерями в скорости.

## 7. Введение в кодирование для каналов с шумом

### 7.1. Две противоположные задачи
| Кодирование источника | Кодирование канала |
|----------------------|--------------------|
| Удаляет избыточность | Вносит избыточность |
| Сжатие данных | Защита от ошибок |

**Раздельный подход:** сначала сжать (убрать избыточность), затем добавить защиту. Это более эффективно, чем пытаться использовать естественную избыточность источника.

### 7.2. Постановка задачи помехоустойчивого кодирования
- Вход канала: двоичные символы $\{0,1\}$.
- Выход канала: также двоичные символы, но возможны ошибки.
- Цель: обеспечить безошибочную передачу (или уменьшить вероятность ошибки до требуемого уровня).

## 8. Простейшие корректирующие коды

### 8.1. Расстояние Хэмминга
Расстояние Хэмминга $d(x,y)$ — количество позиций, в которых последовательности $x$ и $y$ различаются.

### 8.2. Пример 4.1 (троекратный повтор)
- Кодовые слова: $000$ (для 0) и $111$ (для 1).
- Декодирование: выбираем ближайшее по Хэммингу кодовое слово (решение по большинству).
- Исправляет все одиночные ошибки.
- Скорость кода: $R = 1/3 \approx 0.33$ бит/символ канала.

### 8.3. Пример 4.2 (более сложный)
- Сообщения: 00, 01, 10, 11.
- Кодовые слова длины 5 (см. таблицу в лекции).
- Скорость: $R = 2/5 = 0.4$ бит/символ канала.
- Также исправляет все одиночные ошибки.

**Вывод:** Более длинные коды могут иметь более высокую скорость при той же корректирующей способности.

## Итоги лекции

1. **Избыточность кода Хаффмена** оценивается теоремой Галлагера, но даже она может быть велика для малых $H$.
2. **Код Шеннона** — простой неоптимальный код, предшественник арифметического кодирования.
3. **FV-кодирование** позволяет теоретически достичь скорости $R \to H$, но прямое применение побуквенных методов к блокам невозможно из-за экспоненциального роста сложности.
4. **Арифметическое кодирование** решает проблему сложности, обеспечивая $\bar{R} \to H$ с вычислительными затратами $\Theta(n^2)$ или даже $\Theta(n)$.
5. **Помехоустойчивое кодирование** вносит избыточность для защиты от ошибок. Простейший способ — повторение битов (код с повторением).
6. **Расстояние Хэмминга** — ключевое понятие для оценки корректирующей способности кода.
