### 1. Общая схема процесса обработки информации. Основные виды и процедуры обработки информации.

**Ответ:**

Процесс обработки информации — это преобразование исходных данных в форму, пригодную для принятия решений.

**Общая схема:**
```
Исходные данные → Предобработка → Извлечение признаков → Классификация/Кластеризация → Интерпретация
```

1. **Сбор данных:** получение сырых сигналов, изображений, текстов.
2. **Предобработка:** устранение шума, нормализация, заполнение пропусков.
3. **Извлечение признаков:** преобразование сырых данных в компактное описание (вектор признаков).
4. **Обработка (анализ):** классификация (с учителем), кластеризация (без учителя), поиск закономерностей.
5. **Интерпретация и принятие решения:** выдача результата.

**Основные виды обработки:**
- **Обработка сигналов** (фильтрация, сжатие).
- **Обработка изображений** (сегментация, распознавание).
- **Обработка текстов** (поиск ключевых слов, тональность).
- **Статистический анализ данных** (корреляция, регрессия).

**Процедуры обработки (с точки зрения обучения):**
| Тип обучения | Суть | Пример |
|--------------|------|--------|
| **С учителем** | Есть метки классов, обучаем классификатор | SVM, дерево решений, байесовский классификатор |
| **Без учителя** | Нет меток, ищем группы похожих объектов | K-means, иерархическая кластеризация |
| **Полуконтрольное** | Мало меток + много немаркированных данных | графовые методы |
| **Обучение с подкреплением** | Агент учится на наградах/штрафах | Q-learning |

**Место распознавания образов:** Это ключевая задача обработки информации, которая занимается автоматическим отнесением объекта к одному из заранее известных классов на основе его признаков. Распознавание находится на стыке **статистики**, **машинного обучения** и **искусственного интеллекта**.

### 2. Классификация базовых подходов к обработке информации. Место распознавания образов в ряду технологий обработки информации.

**Ответ:**

**Классификация подходов к обработке информации:**

| Подход                               | Характеристика                                    | Типичные методы                                               |
| ------------------------------------ | ------------------------------------------------- | ------------------------------------------------------------- |
| **Статистический**                   | Используются вероятностные модели данных          | Байесовский классификатор, EM-алгоритм, оценивание плотностей |
| **Детерминистский (геометрический)** | Основан на расстояниях и геометрических свойствах | Метод эталонов, SVM (линейный), K-means                       |
| **Структурный (лингвистический)**    | Объект представляется как комбинация примитивов   | Распознавание иерархических структур, грамматики              |
| **Нейросетевой**                     | Моделируются биологические нейронные сети         | MLP, CNN, RNN                                                 |
| **Логический (символьный)**          | Используются правила и предикаты                  | Деревья решений, логические правила                           |

**Место распознавания образов:**
Распознавание образов является **центральной задачей** обработки информации, поскольку многие другие задачи (классификация, диагностика, предсказание) сводятся к ней. Оно связывает:
- **Теорию вероятностей и статистику** (байесовский подход, оценивание параметров)
- **Математический анализ** (оптимизация, градиентные методы)
- **Линейную алгебру** (матрицы, собственные вектора, ядра)
- **Геометрию** (расстояния, гиперплоскости)

**Пример иерархии:**
```
Обработка информации
├── Низкоуровневая (фильтрация, сжатие)
├── Среднеуровневая (выделение признаков, сегментация)
├── Высокоуровневая (распознавание, принятие решений) ← Распознавание образов
└── Когнитивная (понимание, семантика)
```

### 3. Байесовская теория решений. Решающее правило на основе минимизации условного риска.

**Ответ:**

**Суть:** Мы хотим принимать решения (относить объект к классу) так, чтобы **минимизировать средние потери** от ошибок. При этом разные ошибки могут стоить по-разному.

**Исходные данные:**
- $M$ классов: $\omega_1, ..., \omega_M$.
- Априорные вероятности: $P(\omega_j)$ — как часто встречается класс.
- Функции правдоподобия: $p(x|\omega_j)$ — распределение признаков внутри класса.
- Платёжная матрица: $\lambda_{ij} = \lambda(\alpha_i|\omega_j)$ — стоимость решения $\alpha_i$, когда истинный класс $\omega_j$.

**Условный риск** — средние потери для данного $x$ при решении $\alpha_i$:
$$r(\alpha_i|x) = \sum_{j=1}^{M} \lambda_{ij} P(\omega_j|x)$$

**Оптимальное правило (минимум условного риска — МУР):**
Для каждого $x$ выбираем решение $\alpha_i$, которое минимизирует $r(\alpha_i|x)$:
$$\omega_i : r(\alpha_i|x) \le r(\alpha_j|x) \quad \forall j \ne i$$

**Для двух классов ($M=2$) решающее правило через отношение правдоподобия:**
$$l(x) = \frac{p(x|\omega_1)}{p(x|\omega_2)} \mathrel{\mathop{\gtrless}^{\omega_1}_{\omega_2}} l_0, \quad l_0 = \frac{(\lambda_{12} - \lambda_{22}) P(\omega_2)}{(\lambda_{21} - \lambda_{11}) P(\omega_1)}$$

**Смысл порога $l_0$:**
- Если штрафы за ошибки одинаковы ($\lambda_{12} = \lambda_{21} = 1$, $\lambda_{11} = \lambda_{22} = 0$), то $l_0 = P(\omega_2)/P(\omega_1)$.
- Если, вдобавок, классы равновероятны ($P(\omega_1) = P(\omega_2)$), то $l_0 = 1$.

**Итог:** МУР позволяет учитывать **разную стоимость ошибок** (например, спутать рак с простудой намного дороже, чем спутать грипп с простудой).

### 4. Байесовская теория решений. Решающие правила на основе максимума апостериорной вероятности и функции правдоподобия.

**Ответ:**

Эти правила являются частными случаями МУР при упрощающих предположениях.

#### 4.1. Критерий максимума апостериорной вероятности (МАВ)

**Предположение:** все ошибки одинаково плохи, правильные решения не штрафуются:
$$\lambda_{ij} = \begin{cases} 0, & i=j \\ 1, & i \ne j \end{cases}$$

**Тогда условный риск:** $r(\alpha_i|x) = 1 - P(\omega_i|x)$.

**Минимизация риска эквивалентна максимизации апостериорной вероятности:**
$$\omega_i : P(\omega_i|x) \ge P(\omega_j|x) \quad \forall j \ne i$$

**Через формулу Байеса:** $P(\omega_i|x) = \dfrac{p(x|\omega_i)P(\omega_i)}{p(x)}$, поэтому достаточно сравнивать произведения $p(x|\omega_i)P(\omega_i)$.

**Для двух классов:** $l(x) = \dfrac{p(x|\omega_1)}{p(x|\omega_2)} \mathrel{\mathop{\gtrless}^{\omega_1}_{\omega_2}} \dfrac{P(\omega_2)}{P(\omega_1)}$.

**Применение:** Минимизирует **суммарную вероятность ошибки**. Часто называется **критерием идеального наблюдателя**.

#### 4.2. Критерий максимума правдоподобия (МП)

**Дополнительное предположение:** априорные вероятности классов равны (или неизвестны): $P(\omega_i) = 1/M$.

**Тогда МАВ сводится к сравнению правдоподобий:**
$$\omega_i : p(x|\omega_i) \ge p(x|\omega_j) \quad \forall j \ne i$$

**Для двух классов:** $l(x) = \dfrac{p(x|\omega_1)}{p(x|\omega_2)} \mathrel{\mathop{\gtrless}^{\omega_1}_{\omega_2}} 1$.

**Применение:** Используется, когда ничего не известно о частотах классов и штрафах. Требует только знания $p(x|\omega_i)$.

### 5. Понятие разделяющих функций. Обобщённая структура решающего правила.

**Ответ:**

**Разделяющая функция** $g_i(x)$ — это числовая функция, значение которой для класса $\omega_i$ тем больше, чем "увереннее" классификатор относит объект $x$ к этому классу.

**Обобщённое решающее правило:**
$$\omega_i : g_i(x) \ge g_j(x) \quad \forall j \ne i$$

То есть объект относится к классу с **максимальным** значением разделяющей функции.

**Соответствие с ранее изученными критериями:**
| Критерий | Разделяющая функция $g_i(x)$ |
|----------|------------------------------|
| МУР | $-r(\alpha_i|x)$ (берём с минусом, т.к. риск нужно минимизировать) |
| МАВ | $P(\omega_i|x)$ или $P(\omega_i) p(x|\omega_i)$ |
| МП | $p(x|\omega_i)$ |

**Полезное преобразование:** Логарифмирование (монотонная функция) не меняет максимума, но упрощает вычисления:
$$g_i'(x) = \ln(g_i(x))$$

**Для двух классов** часто вводят **одну** разделяющую функцию:
$$g(x) = g_1(x) - g_2(x), \quad \omega_1: g(x) > 0, \quad \omega_2: g(x) < 0$$

**Обобщённая структура классификатора (рис. из лекции 3):**
```
Вход x → Вычисление g₁(x), g₂(x), ..., g_M(x) → Выбор argmax → Класс ω_i
```

**Преимущество:** Единый подход для всех байесовских классификаторов. Достаточно определить $g_i(x)$ — и правило готово.

### 6. Распознавание образов, описываемых гауссовскими векторами с различными математическими ожиданиями.

**Ответ:**

**Постановка:** Признаки $x$ распределены нормально, ковариационные матрицы **одинаковы** для всех классов ($C_i = C$), но математические ожидания $m_i$ разные.

**Почему это важно:** Это самый простой и популярный частный случай гауссовских распределений.

#### 6.1. Разделяющая функция (логарифмическая форма)

$$g_i'(x) = -\frac{1}{2}(x - m_i)^T C^{-1}(x - m_i) + \ln P(\omega_i)$$

Член $-\frac{1}{2}\ln|C_i|$ одинаков для всех классов и опускается.

#### 6.2. Линейность

Раскрывая скобки, получаем **линейную** разделяющую функцию:
$$g_i'(x) = x^T C^{-1} m_i - \frac{1}{2} m_i^T C^{-1} m_i + \ln P(\omega_i)$$

**Вывод:** классификатор — **линейный**.

#### 6.3. Для двух классов

Правило сравнения отношений правдоподобия после логарифмирования даёт:
$$g''(x) = x^T C^{-1}(m_1 - m_2) - \frac{1}{2}(m_1 + m_2)^T C^{-1}(m_1 - m_2) \mathrel{\mathop{\gtrless}^{\omega_1}_{\omega_2}} l_0'$$

Граница — **гиперплоскость**.

#### 6.4. Расчёт ошибок

Величина $g''(x)$ при гипотезах $\omega_1$ и $\omega_2$ распределена нормально с параметрами:
- $m_{g1} = h$, $D_{g1} = 2h$
- $m_{g2} = -h$, $D_{g2} = 2h$

где $h = \frac{1}{2}(m_1 - m_2)^T C^{-1}(m_1 - m_2)$ — **половина квадрата расстояния Махаланобиса**.

**Вероятности ошибок** (при $l_0' = 0$, т.е. $P(\omega_1) = P(\omega_2)$):
$$\alpha = \beta = \Phi\left(-\frac{1}{2}\sqrt{(m_1 - m_2)^T C^{-1}(m_1 - m_2)}\right)$$

**Ключевой вывод:** Качество разделения определяется только **расстоянием Махаланобиса** между центрами классов.

### 7. Распознавание образов, описываемых гауссовскими векторами с различными ковариационными матрицами.

**Ответ:**

**Постановка:** Признаки $x$ распределены нормально, но ковариационные матрицы $C_i$ **разные** у разных классов. Математические ожидания $m_i$ тоже могут быть разными.

#### 7.1. Разделяющая функция

$$g_i'(x) = -\frac{1}{2}\ln|C_i| - \frac{1}{2}(x - m_i)^T C_i^{-1}(x - m_i) + \ln P(\omega_i)$$

В отличие от предыдущего случая, член $-\frac{1}{2}\ln|C_i|$ уже **не одинаков** для всех классов и не может быть отброшен.

#### 7.2. Квадратичность

Раскрывая скобки, получаем **квадратичную** форму:
$$g_i'(x) = -\frac{1}{2} x^T C_i^{-1} x + x^T C_i^{-1} m_i - \frac{1}{2} m_i^T C_i^{-1} m_i - \frac{1}{2}\ln|C_i| + \ln P(\omega_i)$$

**Вывод:** классификатор — **квадратичный**. Граница между классами — поверхность второго порядка (эллипсоид, гиперболоид, параболоид).

#### 7.3. Примеры границ из лекции 4 (рис. 4):
- **Окружность:** центры классов совпадают, дисперсии разные.
- **Парабола:** один класс "обнимает" другой.
- **Гипербола:** области решений могут быть разрывными.

#### 7.4. Расчёт ошибок

В отличие от случая равных ковариаций, $g''(x)$ **не является гауссовской** (это нелинейная функция от $x$). Поэтому:
- Точный расчёт сложен (требует численного интегрирования).
- Используют **гауссовскую аппроксимацию** (при большом $n$).
- Или **статистическое моделирование** (надёжный практический метод).

**Вывод:** Разные ковариации дают более гибкие (квадратичные) границы, но усложняют анализ ошибок.

### 8. Распознавание образов, описываемых произвольными законами распределения.

**Ответ:**

**Проблема:** В реальной жизни данные не обязательно гауссовские. Вид распределения может быть неизвестен или очень сложным.

#### 8.1. Основные подходы к решению

| Подход                                     | Идея                                          | Когда применять                           |                                             |
| ------------------------------------------ | --------------------------------------------- | ----------------------------------------- | ------------------------------------------- |
| **Наивный байесовский классификатор**      | Предполагаем независимость признаков          | Признаки почти независимы или данных мало |                                             |
| **Непараметрическое оценивание плотности** | Восстанавливаем $p(x                          | \omega_i)$ по выборке (Парзен, k-NN)      | Распределение неизвестно, данных достаточно |
| **Параметрическая модель + оценивание**    | Задаём вид распределения, оцениваем параметры | Есть обоснованное предположение о виде    |                                             |

#### 8.2. Наивный байесовский классификатор (Naive Bayes)

**Допущение:** признаки статистически независимы внутри каждого класса.

$$p(x|\omega_i) = \prod_{k=1}^{n} p(x_k|\omega_i)$$

**Преимущества:**
- Простота обучения (оцениваем одномерные плотности).
- Работает даже при малых выборках.
- Может обрабатывать смешанные типы признаков.

**Недостаток:** Если признаки сильно зависимы, качество может быть низким.

#### 8.3. Логарифм отношения правдоподобия для двух классов

$$g''(x) = \sum_{k=1}^{n} \ln\frac{p(x_k|\omega_1)}{p(x_k|\omega_2)}$$

По центральной предельной теореме при большом $n$ эта сумма приближается к нормальному распределению, что позволяет оценивать ошибки.

**Вывод:** Для произвольных распределений нет единой "волшебной" формулы, но есть набор мощных практических методов, каждый из которых хорош в своей области.

### 9. Распознавание образов, описываемых бинарными признаками.

**Ответ:**

**Постановка:** Каждый признак $x_k$ принимает только два значения: 0 или 1 (да/нет, истина/ложь, наличие/отсутствие). Это важный частный случай дискретных признаков.

#### 9.1. Модель

Вероятностное описание класса $\omega_i$ задаётся набором вероятностей:
$$p_{k,i} = P(x_k = 1 | \omega_i), \quad 1 - p_{k,i} = P(x_k = 0 | \omega_i)$$

При наивном предположении о независимости признаков:
$$p(x|\omega_i) = \prod_{k=1}^{n} p_{k,i}^{x_k} (1-p_{k,i})^{1-x_k}$$

#### 9.2. Решающее правило для двух классов

Логарифм отношения правдоподобия:
$$g''(x) = \sum_{k=1}^{n} \left[ x_k \ln\frac{p_k}{q_k} + (1-x_k) \ln\frac{1-p_k}{1-q_k} \right]$$

После преобразований получаем **линейное** решающее правило:
$$\sum_{k=1}^{n} \alpha_k x_k + \alpha_0 \mathrel{\mathop{\gtrless}^{\omega_1}_{\omega_2}} 0$$

где $\alpha_k = \ln\frac{p_k(1-q_k)}{q_k(1-p_k)}$, $\alpha_0 = \sum_{k=1}^{n} \ln\frac{1-p_k}{1-q_k} - l_0'$.

#### 9.3. Частный случай: одинаковые вероятности для всех признаков

Если $p_k = p$, $q_k = q$ для всех $k$, то правило упрощается до сравнения количества единиц $L_x = \sum x_k$ с порогом:
$$L_x \mathrel{\mathop{\gtrless}^{\omega_1}_{\omega_2}} L_0$$

Это позволяет точно рассчитать ошибки через биномиальное распределение:
$$\alpha = \sum_{i=0}^{L_0} C_N^i q^i (1-q)^{N-i}, \quad \beta = \sum_{i=L_0+1}^{N} C_N^i p^i (1-p)^{N-i}$$

**Пример применения:** распознавание бинарных изображений (например, букв) при искажениях (шум инвертирует пиксели с вероятностью $p$).

### 10. Основы теории оценивания. Параметрическое и непараметрическое оценивание в статистических моделях данных.

**Ответ:**

**Оценивание** — это определение неизвестных параметров или характеристик распределения по обучающей выборке.

#### 10.1. Параметрическое оценивание

**Что известно:** вид распределения $p(x|\theta)$ (например, нормальное, экспоненциальное, биномиальное). **Неизвестны** параметры $\theta$ (например, $m$, $C$, $p$).

**Задача:** по выборке $X^N = \{x^{(1)}, ..., x^{(N)}\}$ найти оценку $\hat{\theta} = \hat{\theta}(X^N)$.

**Примеры:**
- Для нормального распределения: $\hat{m} = \frac{1}{N}\sum x^{(i)}$ (выборочное среднее).
- Для схемы Бернулли: $\hat{p} = u/N$ (частота события).

#### 10.2. Непараметрическое оценивание

**Что неизвестно:** вообще всё — ни вид распределения, ни параметры. Есть только выборка.

**Задача:** восстановить саму функцию плотности $p(x)$.

**Примеры методов:**
- Гистограмма
- Метод Парзена (ядерное оценивание)
- Метод $k$-ближайших соседей

#### 10.3. Сравнение

| Характеристика | Параметрическое | Непараметрическое |
|----------------|-----------------|-------------------|
| **Априорная информация** | Вид распределения известен | Ничего не известно |
| **Объём выборки** | Небольшой (оцениваем мало чисел) | Большой (восстанавливаем функцию) |
| **Сложность** | Низкая | Высокая |
| **Риск** | При неверном предположении о виде — большая ошибка | Меньше риск, но нужны данные |
| **Пример** | Оценка среднего по выборке | Оценка плотности методом Парзена |

#### 10.4. Свойства «хороших» оценок

| Свойство | Смысл |
|----------|-------|
| **Несмещённость** | $M[\hat{\theta}] = \theta$ (нет систематической ошибки) |
| **Состоятельность** | $\hat{\theta} \xrightarrow{P} \theta$ при $N \to \infty$ (с ростом данных точность растёт) |
| **Эффективность** | Минимальная дисперсия среди всех несмещённых оценок |
| **Робастность** | Устойчивость к выбросам |

### 11. Параметрическое обучение в задачах распознавания. Метод максимума правдоподобия (МП).

**Ответ:**

**Идея:** Выбираем такие значения параметров $\theta$, при которых наблюдаемая обучающая выборка **наиболее вероятна**.

#### 11.1. Функция правдоподобия

Для независимой выборки:
$$P(X^N|\theta) = \prod_{i=1}^{N} p(x^{(i)}|\theta)$$

**Оценка МП:**
$$\hat{\theta} = \arg\max_{\theta} P(X^N|\theta)$$

Удобнее работать с **логарифмической функцией правдоподобия** (сумма вместо произведения):
$$\ln P(X^N|\theta) = \sum_{i=1}^{N} \ln p(x^{(i)}|\theta)$$

#### 11.2. Нахождение оценки

Если функция дифференцируема, решаем систему уравнений:
$$\frac{\partial \ln P(X^N|\theta)}{\partial \theta_j} = 0, \quad j = 1..h$$

#### 11.3. Примеры

**Пример 1: Оценка среднего для нормального распределения ($C$ известна)**
$$\hat{m} = \frac{1}{N} \sum_{i=1}^{N} x^{(i)}$$

**Пример 2: Оценка вероятности успеха в схеме Бернулли**
$$\hat{p} = \frac{u}{N}, \quad u = \sum_{i=1}^{N} x^{(i)} \quad (x^{(i)} \in \{0,1\})$$

**Пример 3: Оценка среднего и ковариации нормального распределения**
$$\hat{m} = \frac{1}{N} \sum_{i=1}^{N} x^{(i)}, \quad \tilde{C} = \frac{1}{N} \sum_{i=1}^{N} (x^{(i)} - \hat{m})(x^{(i)} - \hat{m})^T$$

#### 11.4. Свойства оценок МП
- **Асимптотически несмещённые** (при $N \to \infty$ смещение стремится к 0)
- **Состоятельные**
- **Эффективные** (дисперсия минимальна)
- **Асимптотически нормальные**

**Недостаток:** могут быть смещёнными при малых $N$ (например, $\tilde{C}$ смещена; несмещённая — с делителем $N-1$).

#### 11.5. Использование в распознавании (подстановочные алгоритмы, plug-in)

Оценки МП подставляются в плотности распределения:
$$\tilde{p}(x|\omega_i) = p(x|\hat{\theta}_i, \omega_i)$$

Затем используется байесовское решающее правило.

### 12. Параметрическое обучение в задачах распознавания. Метод максимума апостериорной вероятности (МАВ).

**Ответ:**

**Идея:** Параметр $\theta$ рассматривается как случайная величина с известным **априорным распределением** $p(\theta)$. Мы ищем значение $\theta$, которое максимизирует **апостериорную плотность** $p(\theta|X^N)$.

#### 12.1. Формула Байеса для параметра

$$p(\theta|X^N) = \frac{P(X^N|\theta) p(\theta)}{P(X^N)}$$

**Оценка МАВ:**
$$\hat{\theta}_{MAP} = \arg\max_{\theta} p(\theta|X^N) = \arg\max_{\theta} \left[ P(X^N|\theta) p(\theta) \right]$$

(знаменатель $P(X^N)$ не зависит от $\theta$, его можно опустить).

#### 12.2. Связь с методом МП

- Если априорное распределение **равномерное** ($p(\theta) = const$), то МАВ совпадает с МП.
- Если априорное распределение **информативное**, МАВ даёт "сглаженную" оценку, которая может быть лучше при малых выборках.

#### 12.3. Пример: оценка среднего нормального распределения с известной ковариацией

**Априорное распределение:** $p(m) = N(m, m_0, C_0)$.

**Апостериорное распределение** (тоже нормальное) имеет параметры:
$$m_N = C_N (N C^{-1} \tilde{m} + C_0^{-1} m_0), \quad C_N = (N C^{-1} + C_0^{-1})^{-1}$$

**Анализ:**
- $m_N$ — взвешенное среднее между выборочным средним $\tilde{m}$ и априорным $m_0$.
- При $N \to \infty$: $m_N \to \tilde{m}$ (влияние априорных знаний исчезает).
- При малых $N$: априорное знание "подтягивает" оценку к $m_0$.

**Оценка МАВ:** $\hat{m}_{MAP} = m_N$.

#### 12.4. Сравнение МП и МАВ (по данным из лекции 6)

| Условие ($C=1$) | Дисперсия ошибки (МП) | Дисперсия ошибки (МАВ) | Выигрыш           |
| --------------- | --------------------- | ---------------------- | ----------------- |
| $C_0=1$         | 0.125                 | 0.111                  | есть              |
| $C_0=0.25$      | 0.125                 | 0.083                  | **значительный**  |
| $C_0=0.1$       | 0.125                 | 0.056                  | **очень большой** |

**Вывод:** Чем точнее априорное знание (меньше $C_0$), тем больше выигрыш МАВ.

### 13. Непараметрическое обучение в задачах распознавания. Метод Парзена.

**Ответ:**

**Проблема:** Ни вид распределения, ни параметры неизвестны. Есть только обучающая выборка.

**Идея Парзена:** Оценить плотность $p(x)$ как сумму "вкладов" от всех точек выборки, где вклад каждой точки имеет форму "колокола" (ядра).

#### 13.1. Одномерный случай

$$\tilde{p}(x) = \frac{1}{Nh} \sum_{i=1}^{N} \varphi\left(\frac{x - x^{(i)}}{h}\right)$$

где:
- $\varphi(u)$ — **ядро** (функция, обычно симметричная, с максимумом в 0)
- $h$ — **ширина окна** (сглаживающий параметр)

#### 13.2. Примеры ядер

| Название | Формула $\varphi(u)$ |
|----------|----------------------|
| Прямоугольное | $1/2$ при $|u| \le 1$, иначе 0 |
| Треугольное | $1 - |u|$ при $|u| \le 1$ |
| Гауссовское | $\frac{1}{\sqrt{2\pi}} e^{-u^2/2}$ |
| Экспоненциальное | $\frac{1}{2} e^{-|u|}$ |

#### 13.3. Выбор ширины окна $h$ — критический момент

**Условия сходимости:**
$$\lim_{N \to \infty} h(N) = 0, \quad \lim_{N \to \infty} N h(N) = \infty$$

**Практическая формула:** $h(N) = N^{-r/n}$, $0 < r < 1$.

**Баланс:**
- $h$ слишком мало → оценка "дерганая" (большая дисперсия).
- $h$ слишком велико → оценка слишком гладкая (большое смещение).

#### 13.4. Многомерный случай

$$\tilde{p}(x) = \frac{1}{N h^n} \sum_{i=1}^{N} \varphi\left(\frac{x - x^{(i)}}{h}\right)$$

#### 13.5. Использование в распознавании

Для двух классов (при неизвестных априорных вероятностях):
$$\frac{N_1}{N} \tilde{p}(x|\omega_1) \mathrel{\mathop{\gtrless}^{\omega_1}_{\omega_2}} \frac{N_2}{N} \tilde{p}(x|\omega_2)$$

**Преимущества:** Гибкость, сходимость к истинной плотности при $N \to \infty$.
**Недостатки:** Нужно хранить всю выборку, выбор $h$ не тривиален, "проклятие размерности".

### 14. Непараметрическое обучение в задачах распознавания. Метод $k$-ближайших соседей ($k$-NN).

**Ответ:**

**Идея:** Вместо фиксированной ширины окна $h$, берём переменный объём, содержащий ровно $k$ ближайших соседей.

#### 14.1. Оценка плотности

Для точки $x$:
1. Находим $k$ ближайших соседей из обучающей выборки.
2. Определяем радиус $r(x)$ — расстояние до $k$-го соседа.
3. Вычисляем объём гипершара: $V_n = \frac{2 r^n \pi^{n/2}}{n \Gamma(n/2)}$.
   - $n=1$: $V_1 = 2r$
   - $n=2$: $V_2 = \pi r^2$
   - $n=3$: $V_3 = \frac{4}{3}\pi r^3$
4. Оценка плотности: $\tilde{p}(x) = \dfrac{k}{N V_n}$.

#### 14.2. Условия состоятельности

$$\lim_{N \to \infty} k(N) = \infty, \quad \lim_{N \to \infty} \frac{k(N)}{N} = 0$$

**Рекомендации:** $k = a N^\gamma$ ($0 < \gamma < 1$) или $k = b \ln(N+1)$.

#### 14.3. Решающее правило для распознавания

Если в гипершаре радиуса $r(x)$ оказалось $k_i$ точек класса $\omega_i$ ($\sum k_i = k$), то:
$$k_i \mathrel{\mathop{\gtrless}^{\omega_i}_{\omega_j}} k_j$$

**Для двух классов:** $k_1 > k_2 \to \omega_1$, иначе $\omega_2$.

**Важно:** Метод $k$-NN является **метрическим классификатором** и даёт простую интерпретацию: относим объект к тому классу, чьих представителей большинство среди $k$ соседей.

#### 14.4. Свойства

- **Ошибка:** $E_s \le \tilde{E}_s \le 2E_s$ (ошибка $k$-NN не более чем вдвое хуже байесовской!).
- **Преимущества:** Простота, отсутствие необходимости подбирать $h$, адаптация к локальной плотности.
- **Недостатки:** Нужно хранить всю выборку, вычислительная сложность, проклятие размерности.

### 15. Распознавание образов с помощью функций расстояния.

**Ответ:**

**Основная идея (геометрический подход):** Классификация основана на близости объектов в пространстве признаков. Объект относится к тому классу, чей эталон (или группа эталонов) находится к нему ближе всего.

#### 15.1. Функции расстояния (метрики)

**Евклидово расстояние:**
$$d_E(x, z) = \sqrt{(x - z)^T (x - z)}$$

**Расстояние Махаланобиса:**
$$d_M(x, z) = \sqrt{(x - z)^T C^{-1} (x - z)}$$
Учитывает корреляцию признаков.

**$l_p$-норма Минковского:**
$$d_p(x, z) = \left( \sum_{s=1}^{n} |x_s - z_s|^p \right)^{1/p}$$

#### 15.2. Случай одного эталона на класс

**Правило:** $\omega_i : d(x, z^{(i)}) \le d(x, z^{(j)}) \quad \forall j \ne i$.

**Связь с байесовским подходом:** При использовании евклидова расстояния разделяющие функции становятся **линейными**:
$$g^{(i)}(x) = x^T z^{(i)} - \frac{1}{2} z^{(i)T} z^{(i)}$$

Граница между классами — гиперплоскость, перпендикулярная отрезку, соединяющему эталоны.

#### 15.3. Случай нескольких эталонов на класс

Расстояние до класса: $d(x, Z^{(i)}) = \min_{k} d(x, z^{(i,k)})$.

**Правило:** относим $x$ к тому классу, до которого расстояние минимально.

**Связь с $k$-NN:** Метод эталонов — частный случай $k$-NN ($k=1$) с отобранными эталонами вместо всей выборки.

#### 15.4. Обучение метрических алгоритмов (выбор эталонов)

Алгоритмы типа **STOLP** и **FRiS-STOLP** позволяют отобрать из обучающей выборки наиболее информативные эталоны, отбросив шумовые и периферийные объекты.

#### 15.5. Бинарные признаки

Специальные меры близости:
- Нормированное скалярное произведение: $d(x,z) = \frac{x^T z}{\sqrt{(x^T x)(z^T z)}}$
- Мера Танимото: $d(x,z) = \frac{x^T z}{x^T x + z^T z - x^T z}$

### 16. Метод опорных векторов (SVM).

**Ответ:**

**Главная идея:** SVM ищет разделяющую гиперплоскость с **максимальным зазором** (margin). Это повышает обобщающую способность.

#### 16.1. Линейно разделимый случай (жёсткий зазор)

Гиперплоскость: $g(x) = w^T x - b_0 = 0$.

Условие разделимости: $d^{(i)} (w^T x^{(i)} - b_0) \ge 1$ (после нормировки).

Ширина полосы: $\Delta = 2 / \|w\|$.

**Задача оптимизации:**
$$\begin{cases} \frac{1}{2} w^T w \to \min \\ d^{(i)}(w^T x^{(i)} - b_0) \ge 1, \quad i = 1..N \end{cases}$$

**Решение через множители Лагранжа:** $w = \sum_{i=1}^{N} \lambda_i d^{(i)} x^{(i)}$.

Точки с $\lambda_i > 0$ — **опорные векторы**. Только они определяют решение!

#### 16.2. Линейно неразделимый случай (мягкий зазор)

Вводим штрафные переменные $\xi_i \ge 0$:
$$d^{(i)}(w^T x^{(i)} - b_0) \ge 1 - \xi_i$$

**Новая задача:**
$$\begin{cases} \frac{1}{2} w^T w + C \sum_{i=1}^{N} \xi_i \to \min \\ d^{(i)}(w^T x^{(i)} - b_0) \ge 1 - \xi_i, \quad \xi_i \ge 0 \end{cases}$$

$C$ — параметр регуляризации (компромисс между шириной полосы и ошибками).

#### 16.3. Типы опорных векторов в мягком варианте

| Тип | $\lambda_i$ | Положение |
|-----|-------------|------------|
| Периферийные | 0 | Далеко от границы, не влияют |
| Опорные граничные | $0 < \lambda_i < C$ | Лежат точно на границе полосы |
| Опорные нарушители | $\lambda_i = C$ | Внутри полосы или ошибочные |

#### 16.4. Нелинейный SVM (ядро)

Используем ядерный трюк: заменяем скалярное произведение на ядро $K(x, z) = \varphi(x)^T \varphi(z)$.

**Нелинейное решающее правило:**
$$G(x) = \text{sign}\left( \sum_{i=1}^{N} \lambda_i d^{(i)} K(x, x^{(i)}) - b_0 \right)$$

**Популярные ядра:** линейное, полиномиальное, RBF (гауссовское), сигмоидное.

#### 16.5. Мультиклассовый SVM

Схемы: "один против всех" ($M$ классификаторов) или "попарно" ($M(M-1)/2$ классификаторов с голосованием).

#### 16.6. Достоинства и недостатки

| Достоинства | Недостатки |
|-------------|-------------|
| Высокое качество | Чувствительность к выбросам |
| Устойчивость к переобучению (за счёт зазора) | Трудный выбор $C$ и параметров ядра |
| Работает с большими размерностями | Медленное обучение ($O(N^3)$) |
| Компактное решение (опорные векторы) | Результат плохо интерпретируем |

### 17. Кластеризация образов. Алгоритм $K$-средних ($K$-means).

**Ответ:**

**Постановка:** Имеется неразмеченная выборка $X^N$. Нужно разбить её на $K$ кластеров (групп близких объектов). $K$ считается известным.

#### 17.1. Идея алгоритма

Минимизировать **внутриклассовый разброс** (сумму квадратов расстояний от точек до центров кластеров):
$$E_W = \sum_{i=1}^{K} \sum_{x \in X^{N_i}} \|x - m_i\|^2$$

#### 17.2. Шаги алгоритма

1. **Инициализация:** выбрать $K$ начальных центров $m_i$ (случайно или максиминным способом).
2. **Разбиение:** отнести каждую точку к ближайшему центру.
3. **Пересчёт центров:** $m_i = \frac{1}{N_i} \sum_{x \in X^{N_i}} x$ (среднее арифметическое точек кластера).
4. **Проверка сходимости:** если центры изменились больше порога $\delta_0$, вернуться к шагу 2.

#### 17.3. Свойства

- $E_W$ монотонно не возрастает на каждой итерации.
- Алгоритм сходится за конечное число шагов.
- Сходится к **локальному** минимуму (зависит от начальных центров).
- **Чувствителен к выбросам.**

#### 17.4. Как повысить устойчивость

- Запустить алгоритм несколько раз с разными начальными центрами.
- Выбрать разбиение с наименьшим $E_W$.

#### 17.5. Силуэт (Silhouette) — оценка качества кластеризации

Для каждого объекта $x$:
- $a$ — среднее расстояние до объектов **своего** кластера
- $b$ — минимальное среднее расстояние до **чужого** кластера
- $s = (b - a) / \max(a, b)$

$s \in [-1, 1]$:
- $s > 0.5$ — объект хорошо "сидит" в своём кластере
- $s \approx 0$ — на границе
- $s < 0$ — скорее относится к другому кластеру

#### 17.6. Достоинства и недостатки

| Достоинства | Недостатки |
|-------------|-------------|
| Прост и быстр ($O(NK)$ на итерацию) | Требует знания $K$ |
| Гарантированная сходимость | Чувствителен к начальным центрам |
| Широко применяется на практике | Не работает с категориальными признаками |
| | Чувствителен к выбросам и шуму |

### 18. Кластеризация образов. Иерархическая процедура группирования.

**Ответ:**

**Идея:** Построить **дерево вложенных кластеров** (дендрограмму), не требуя заранее знать число кластеров.

#### 18.1. Два направления

| Тип | Начало | Движение | Конец |
|-----|--------|----------|-------|
| **Агломеративная (снизу вверх)** | каждый объект — отдельный кластер | объединяем ближайшие пары | один кластер |
| **Дивизимная (сверху вниз)** | один кластер | разделяем на части | каждый объект — отдельный кластер |

На практике чаще используют **агломеративную** кластеризацию.

#### 18.2. Агломеративный алгоритм (шаги)

1. Начало: $M = N$, $X^{N_i} = \{x^{(i)}\}$.
2. Найти пару кластеров с **минимальным расстоянием** между ними.
3. Объединить их в один кластер ($M = M-1$).
4. Повторять шаги 2-3, пока $M = 1$ (или пока не достигнуто нужное число кластеров).

#### 18.3. Функции расстояния между кластерами (linkage)

| Название | Формула | Характер |
|----------|---------|----------|
| Ближайший сосед (single) | $\min d(x^{(i)}, x^{(j)})$ | Образует "цепочки", чувствителен к шуму |
| Дальний сосед (complete) | $\max d(x^{(i)}, x^{(j)})$ | Компактные сферические кластеры |
| Среднее расстояние (average) | $\frac{1}{N_i N_j} \sum_{i,j} d(x^{(i)}, x^{(j)})$ | Компромисс |
| Расстояние между центрами | $d(m_i, m_j)$ | Быстро, но грубо |
| Расстояние Уорда | $\frac{N_i N_j}{N_i+N_j} d(m_i, m_j)^2$ | Минимизирует рост $E_W$ |

#### 18.4. Дендрограмма

График, показывающий иерархию объединений. По вертикали — расстояние объединения (или шаг). Горизонтальные линии — слияния.

**Выбор числа кластеров:** "отрезаем" дендрограмму на нужном уровне (например, по большому скачку расстояния слияния).

#### 18.5. Достоинства и недостатки

| Достоинства | Недостатки |
|-------------|-------------|
| Не требует задания $K$ | Вычислительная сложность $O(N^3)$ (наивная) |
| Даёт полную иерархию | Память $O(N^2)$ |
| Хорошая визуализация (дендрограмма) | Чувствителен к выбору linkage |
| Может работать с любыми функциями расстояния | Трудно интерпретировать при больших $N$ |

### 19. Кластерный анализ в условиях неизвестного числа классов.

**Ответ:**

Это самая сложная задача кластеризации. Основной подход: выполнить кластеризацию при разных $M$ и выбрать наилучший вариант по специальному **индексу качества**.

#### 19.1. Индекс Калинского — Харабаша (CH)

$$I_{CH}(M) = \frac{E_B (N - M)}{E_W (M - 1)}$$

**Идея:** максимизировать отношение межклассового разброса к внутриклассовому. Хорошо работает для сферических кластеров.

#### 19.2. Индекс Хржановского — Лаи (CL)

$$I_{CL}(M) = \left| \frac{\Delta E_W(M)}{\Delta E_W(M+1)} \right|$$

**Идея:** ищем "скачок" $E_W$ (резкое уменьшение). Максимум индекса указывает на оптимальное $M$.

#### 19.3. Индекс Дэвиса — Болдуина (DB)

$$I_{DB}(M) = \frac{1}{M} \sum_{i=1}^{M} \max_{j \ne i} \frac{e_{wi} + e_{wj}}{h_{ij}}$$

где $e_{wi}$ — среднее расстояние до центра внутри кластера, $h_{ij}$ — расстояние между центрами.

**Идея:** минимизировать среднюю "похожесть" между разными кластерами.

#### 19.4. Индекс GAP

$$I_{GAP}(M) = \frac{1}{B} \sum_{k=1}^{B} \log E_W^*(X^N, Q_M) - \log E_W(X^N, Q_M)$$

**Идея:** сравниваем $\log E_W$ на реальных данных с $\log E_W$ на случайных данных (равномерное распределение). Максимальный разрыв (gap) — оптимальное $M$.

**Преимущество:** может оценивать даже $M=1$ (нет ссылочного значения).

#### 19.5. Силуэт (средний)

$$I_{sv} = \frac{1}{N} \sum_{i=1}^{N} s_i$$

**Идея:** максимизируем среднюю величину силуэта (чем ближе к 1, тем лучше).

#### 19.6. Экспериментальные результаты (из лекции, рис. 6)

- При малых $d_m$ (кластеры слабо разделены) лучше работают **EM-алгоритм** и **K-means**.
- При больших $d_m$ (кластеры сильно разделены) **иерархическая кластеризация** обгоняет.
- **EM-алгоритм** показал наилучшие результаты в целом.

**Вывод:** Нет универсального лучшего метода. Выбор зависит от формы кластеров, разделимости, объёма выборки и наличия шума.

### 20. Нелинейные преобразования и спрямляющие пространства.

**Ответ:**

**Проблема:** В исходном пространстве признаков данные могут быть **линейно не разделимы** (например, XOR). Но можно отобразить их в пространство **более высокой размерности**, где они станут линейно разделимыми.

#### 20.1. Теорема Ковера

> Нелинейное преобразование сложной задачи классификации в пространство более высокой размерности повышает вероятность линейной разделимости образов.

Более того, для **любой** конечной обучающей выборки существует преобразование, делающее классы линейно разделимыми.

#### 20.2. Пример: задача XOR

Точки: (0,0) и (1,1) — класс $\omega_1$; (1,0) и (0,1) — класс $\omega_2$.

Преобразование с помощью радиально-базисных функций (РБФ):
$$\varphi_j(x) = \exp\left(-\frac{\|x - c^{(j)}\|^2}{2\sigma^2}\right)$$

Центры $c^{(j)}$ — все точки выборки. В 4-мерном пространстве образы становятся линейно разделимыми.

#### 20.3. Ядерный трюк (kernel trick)

Вместо явного вычисления $\varphi(x)$ (размерность может быть огромной), работаем через **ядро**:
$$K(x, z) = \varphi(x)^T \varphi(z)$$

Многие линейные алгоритмы (SVM, метод эталонов, $k$-NN) могут быть "ядерно расширены" заменой $x^T z$ на $K(x, z)$.

#### 20.4. Популярные ядра

| Ядро | Формула | Особенность |
|------|---------|-------------|
| Линейное | $x^T z$ | Отсутствие преобразования |
| Полиномиальное | $(x^T z + 1)^p$ | Степень $p$ определяет сложность |
| RBF (гауссовское) | $\exp\left(-\frac{\|x-z\|^2}{2\sigma^2}\right)$ | Бесконечномерное, универсальное |
| Сигмоидное | $\tanh(a x^T z + b)$ | Аналог нейронной сети |

#### 20.5. Расстояние через ядро

$$d^2(\varphi(x), \varphi(z)) = K(x, x) - 2K(x, z) + K(z, z)$$

Это позволяет использовать ядра в метрических алгоритмах.

#### 20.6. Метод потенциальных функций (исторически первый)

Каждая точка создаёт вокруг себя потенциал $K(x, x^{(i)})$. Суммарный потенциал класса $i$:
$$K_i(x) = \sum_{x^{(i)} \in \omega_i} K(x, x^{(i)})$$

**Правило:** $K_1(x) > K_2(x) \to \omega_1$, иначе $\omega_2$.

Обучение — итеративное добавление/вычитание весов опорных точек.

**Связь с SVM:** метод потенциальных функций — предшественник SVM.

#### 20.7. Практический вывод

Если данные линейно не разделимы, не усложняйте сам классификатор — **примените ядерный трюк** к уже существующему линейному методу. Это часто даёт значительное улучшение качества.
